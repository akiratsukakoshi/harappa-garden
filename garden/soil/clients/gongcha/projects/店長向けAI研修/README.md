## 提案書 ver.1.0 の要点(2026-06)

**コンセプト** = 「バイブスコーディング」(人×人×AI)。1対1の "バイブ・コーディング"(AIと1人)でなく、複数人の熱量・ノリを交換して発想を増幅する原っぱ流アプローチ。コミュニケーション総量が多いゴンチャに合う。

**先方の課題(2つの両立)** = ①現場をAIで効率化(急成長の店舗オペを軽く・速く)② でも人の魅力は失わない(ゴンチャの強み=人)。

**提案の核** = トップダウン(やらされる)でなく **ボトムアップ**(店長が考える→提案する→つくる)。研修して終わりにせず**現場実装まで**設計。

**全体設計(知る→夢→遊び→活かす→定着)**:
| 回 | 内容 | 形式 |
|---|---|---|
| 第0回 | パイロット & ヒアリング(本部スタッフ部門・巻き込みの土台) | 対面 |
| 第1回 | 夢を膨らませる(AIの今できること) | 講義2h・オンラインOK |
| 第2回 | 遊びを憶える(業務無関係なものを作る=点火) | 半日・リアル |
| 第3回 | 業務に活かす(棚卸し&アイデア→その場でツール化) | 半日〜1日・リアル |
| 第4回 | 定着(下記 A/B/C から選ぶ) | フォローアップ |

**第4回=定着の3案(ご一緒に選ぶ)**:
- **A コンテスト→実装**(審査→選定アイデアを現場実装まで伴走)
- **B 運用・開発の伴走**(定例で継続支援・LINE等で常時アドバイス)
- **C 経営が見るパイロット**(成果と反応を経営が見て次の打ち手を判断)

**対象・進め方** = 直営店長(20代と若い)。STEP1 パイロット数名 → STEP2 全店長へ段階展開。理想は1ヶ月で第0〜3回、第1〜3回を1泊2日合宿に凝縮も可。**費用・日程は固めず方向性と絵の共有が目的**(たたき台)。

**案内人** = 塚越 暁(Facilitator)+ **元沢 信昭/もっちゃん**(Pacer・ボタリズムコーヒーロースター代表・飲食現場を知るプロ)。
