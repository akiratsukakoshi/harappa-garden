# (B) 当月の流れ(シフト中心)

> 毎月の頭・中・末に走るシフト周りのルーチン。`shift_manager` SKILL のタスクと一対一で対応する。

## 全体図

```
当月1日  ─ 前月稼働の確認 + 翌々月シフトアンケート(LINEスタッフG)
            │
当月10日 ─ 翌々月シフト確定
            │
適宜    ─ 翌月の担当プログラムの LINE グループを作成(都度)
            │
月末    ─ 当月のシフト稼働をまとめる
```

## ステップ

### 1. 月初1日 — 前月稼働確認 + 翌々月シフトアンケート

- **時期**: 毎月1日
- **担当**: シフト管理担当(現状: TODO 確認)
- **動き**:
  - 前月の稼働を確認
  - 翌々月分のシフトアンケートをスタッフ LINE グループにシェア
- **参照**: `shift_manager` SKILL のタスク定義

### 2. 月初10日 — 翌々月シフト確定

- **時期**: 毎月10日まで
- **担当**: シフト管理担当
- **動き**: 翌々月分のシフトをアンケート集計から確定
- **アウトプット**:
  - 翌々月の各プログラム現場責任者・フィールドスタッフ・フォトグラファー = 確定
- **参照**: `shift_manager` SKILL

### 3. 翌月担当プログラムの LINE グループ作成

- **時期**: 都度(翌月の運用が始まる前まで)
- **担当**: 各プログラムの現場責任者
- **動き**: プログラム毎にスタッフ・チームが異なるため、開催毎に LINE グループを作る
- **アウトプット**: 各プログラムのスタッフコミュニケーション基盤
- **次の遷移**: 個別開催の運用は [[program-execution]] へ

### 4. 月末 — 当月シフト稼働のまとめ

- **時期**: 月末
- **担当**: シフト管理担当
- **動き**: 当月の稼働実績を集計
- **参照**: `shift_manager` SKILL

## 不変条件(invariants)

- **シフトは「翌々月」を確定させる**(N月の1日には N+2月分のアンケート、10日には N+2月分の確定)
- LINE グループは **プログラム毎** に作る(プログラム毎にチームが組み変わる)

## 種(自律トリガー)候補 — Phase 3 向けメモ

- 毎月1日: 前月稼働サマリ + 翌々月アンケート送信
- 毎月10日: シフト集計通知
- 月末: 稼働サマリ生成
