# (C) プログラム実施フロー

> 個別開催の現場運用。2週前の企画MTGから始まり、当日実施・お礼・振り返り・写真共有まで。

## 全体図

```
開催2週前 ─ 企画MTG(企画担当者 ↔ 現場責任者)
            │
            ↓
            準備(物品・人員・参加者連絡)
            │
            ↓
            体験参加者がいれば個別案内 ─ テンプレあり
            │
            ↓
開催前日 ─ 天気判断
            │
当日   ─ プログラム実行
            │
            ↓
            体験参加者へお礼 ─ テンプレあり
            │
            ↓
            振り返りレポート → Notion(構造既定)
            │
            ↓
            フォトグラファー → Google Photo アルバム → STORES経由で参加者へ
```

## 登場人物

| 役割 | 担い手 |
|---|---|
| **企画担当者** | 当該プログラムの企画を持つ運営4名 or [[junki-iida]] |
| **現場責任者** | 当該開催を引き受ける運営4名のいずれか |
| **フォトグラファー** | 月次シフトで割り当てられた `role: 写真` のスタッフ |
| **現場スタッフ** | 月次シフトで割り当てられたフィールドスタッフ |

## ステップ

### 1. 企画MTG(開催2週前 目安)

- **担当**: 現場責任者(主導)+ 企画担当者
- **動き**: 現場責任者が企画担当者との企画MTGをセットアップし、当日の詳細を確認
- **アウトプット**: 当日運用の認識合わせ

### 2. 準備

- **担当**: 現場責任者
- **動き**:
  - 必要物品の手配
  - スタッフとプログラム LINE グループ(see [[monthly-cycle]])でやり取り
  - 参加者連絡

### 3. 体験参加者への個別案内(該当時)

- **担当**: 現場責任者
- **対象**: 体験参加者(初参加で正会員ではないケース)
- **テンプレ**: あり(将来 SKILL 化候補)

### 4. 天気判断(前日まで)

- **担当**: 現場責任者
- **動き**: 屋外プログラムのため、前日までに開催可否を判断

### 5. プログラム実行

- **担当**: 現場責任者 + 現場スタッフ + フォトグラファー
- **動き**: 当日運用

### 6. 体験参加者へのお礼

- **担当**: 現場責任者
- **対象**: 体験参加者
- **テンプレ**: あり(将来 SKILL 化候補)

### 7. 振り返りレポート → Notion

- **担当**: 現場責任者
- **アウトプット**: Notion「**フィールドレポート**」DB に1レコード追加
- **DB**: [フィールドレポート](https://www.notion.so/5dab98a40ae443849e3804c0b431abe2)(配置: 原っぱ大学運営ページ(NEW) / フィールド関連 / フィールドレポート)
- **data source**: `collection://ffb8cd73-f4e1-471c-bf2e-305532ead0de`
- **既定スキーマ**(2026-05-23 取得):

| プロパティ | 型 | 内容 |
|---|---|---|
| Name | title | レポートタイトル(例: 「6月10日リトル森戸海岸day」) |
| 開催日 | date | 開催日 |
| コース | multi_select | おやこ学部 / こども学部 / おとな学部 / 余の日 / 海DAY / 千葉 |
| 天気 | multi_select | 晴れ🌞 / 曇り⛅ / 雨☔ / 雷⚡ / 風速10-18m / 雪 / 曇り時々雨 |
| 気温 | text |  |
| 業務時間 | text |  |
| 参加組数 | number |  |
| 体験家族数 | text |  |
| 参加スタッフ | text | 当日メンバー(本文 or text) |
| 調理班へのランチオーダー | text | 調理 role 向け(現状運用なし) |
| アルバム | text | Google Photo の共有 URL |

- **コース** の選択肢に `余の日 / 海DAY / 千葉` があり、toC 1-a/b/c 以外のプログラムも同じテンプレで運用されている(workflows 拡張時に確認)
- ページ本体には自由記述で当日の振り返り・写真・参加者の様子等を記録

### 8. 写真アルバム配布

- **担当**: フォトグラファー → 現場責任者 → 参加者
- **動き**:
  1. フォトグラファーが Google Photo アルバムを作成・共有
  2. 受け取った現場責任者が STORES予約 経由で参加者にシェア

## 不変条件(invariants)

- **天気判断は前日まで** に行う(屋外前提)
- **体験参加者の個別フォロー**(案内・お礼)は必須
- **振り返りは Notion** に置く(構造既定)
- **写真は STORES予約 経由**で参加者にシェア(Google Photo 共有リンクを STORES の管理画面から)

## 種(自律トリガー)候補 — Phase 3 向けメモ

- 開催14日前: 企画MTG準備リマインダー(企画担当者・現場責任者宛)
- 開催前日: 天気判断リマインダー + 翌日の運用サマリ通知
- 開催翌日: 体験者お礼テンプレ生成 + 振り返りレポート Notion ページ自動生成
- 写真アルバム共有受信: STORES予約へ転送タスク生成
