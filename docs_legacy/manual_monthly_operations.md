# Monthly UI (プログラムカレンダー新) 操作マニュアル

このドキュメントでは、月次運用シート (`HMC_Monthly_UI` / `プログラムカレンダー新`) を人間が編集する際のルールと注意点を説明します。

## 1. 入力・編集のルール

### OKなこと（DBに保存されます）
*   **各セルへの文字入力**: 「会場」「活動内容」「時間」「スタッフ」など、白い背景のセルは自由に入力・編集可能です。
    *   編集内容は「Sync Monthly -> DB」を実行した際にデータベースに保存されます。
*   **イベントの追加**: 行を追加し、日付を入力して新しいイベントを作成することができます。

### 特殊な列：「運営スケジュール」(C列)
*   **運営スケジュール**: B列（曜日）とD列（会場）の間にある「運営スケジュール」列は、**データベースには保存されません。**
    *   告知の締め切りや作業メモなど、自由にお使いください。
    *   シートを再生成（Reload）すると、この列の内容は消える可能性があります（現在はクリアされます）。一時的なメモとして利用してください。

### NGなこと（データの破損や保存失敗の原因になります）
*   **列の挿入・削除**: A列〜P列（日付〜請求）の並び順はシステムで固定されています。**この間に列を挿入したり、削除したりしないでください。**
    *   列がずれると、データが誤った項目として保存されてしまいます。
*   **「日付」の形式変更**: 日付列を空欄にしたり、無効な日付を入れると正しく処理されません。
*   **「カテゴリ」の入力規則無視**: 「カテゴリ」列はプルダウン（入力規則）で管理されています。リストにない値を無理やり入力するとエラーの原因になる場合があります。

## 2. 書式設定（太字・文字色・背景色）について

### 一時的な変更は「可能」ですが...
*   **可能**: 視認性を上げるために特定のセルの文字を赤くしたり、太字にすることは可能です。
*   **注意**: ただし、**これらの書式設定はデータベースには保存されません。**
    *   システムは「文字データ」のみを保存します。
    *   もし次回、「Generate Monthly (DB -> Monthly)」を実行してシートを再生成した場合、**あなたが設定した書式はすべてリセットされ、システム標準の書式に戻ります。**
    *   したがって、永続的な情報（「重要！」など）は、色ではなく「備考」欄に文字で入力することを推奨します。

## 3. 列の追加について

*   **右側への追加**: O列（請求）よりも右側の列（P列以降）に、独自のメモ用の列を追加することは可能です。
    *   **注意**: ただし、**P列以降に入力したデータはデータベースには一切保存されません。**
    *   シートを再生成すると、これらの列ごと消滅する可能性があります（シート全体がクリアされるため）。
    *   あくまで「その場限りの作業用」としてお使いください。

## まとめ

| 項目 | ルール | 理由・挙動 |
| :--- | :--- | :--- |
| **文字編集** | ◯ 自由 | 白い列は自由入力OK。DBに保存される。 |
| **行追加** | ◯ 可能 | 日付形式 (`YYYY-MM-DD`) を守ること。 |
| **列の挿入** | **× 禁止** | システムが列位置を見失い、データが壊れるため。 |
| **書式変更** | △ 一時的 | DBには保存されない。再生成で消える。 |
| **列追加(右側)** | △ 一時的 | DBには保存されない。再生成で消える。 |
