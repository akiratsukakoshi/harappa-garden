# Letter Opener 操作マニュアル

## 1. 概要
`letter_opener` は、手元に届いた郵便物（封筒、ハガキ等）の写真をAI（Gemini OCR）で自動解析し、HMCパイロット（AIエージェント）と協働して「タスク化」するためのスキルです。
本スキルにより、郵便物の内容確認から期日の把握、タスクデータベース (`backlog.md`) への一元管理までをスムーズに行うことができます。

## 2. 事前準備
1. **Google Driveの用意**
   未処理の画像を入れる `Inbox` フォルダと、処理済み画像を移動させる `Processed` フォルダをGoogle Drive上に作成します。
   （例：`Letters/Inbox`, `Letters/Processed`）
2. **環境変数の設定**
   HMCのルートディレクトリにある `.env` ファイルに、作成したフォルダのIDを設定します。
   ```env
   LETTER_DRIVE_INBOX_ID=your_inbox_folder_id
   LETTER_DRIVE_PROCESSED_ID=your_processed_folder_id
   ```

## 3. 基本的な使い方 (HMCパイロット経由)

本スキルは、基本的にHMCのAIエージェントとのチャットを通じて実行します。

### Step 1: 写真のアップロード
スマートフォンのGoogle Driveアプリなどから、届いた郵便物の写真を撮影し、事前に設定した **Inboxフォルダ** にアップロードします。

### Step 2: 解析の指示
HMCのチャット画面で、AIエージェントに向かって以下のように話しかけます。
* **「郵便物の確認をしよう」**
* **「Letter Openerを実行して」**

### Step 3: 内容の確認と修正
エージェントがDriveから写真を読み込み、AIによる解析結果（タスク内容、種類、期限など）をチャット画面に箇条書きで提示します。
* 内容に問題がなければ、「OK」「その内容で登録して」と返信します。
* 修正箇所があれば、「タスク1の期限を来月末にして」「カテゴリを支払いに変えて」などと指示します。エージェントが修正を行います。

### Step 4: タスクの自動登録
エージェントへの承認が完了すると、自動的に以下の処理が行われます。
1. タスクが `tasks/letter_tasks.md` に一旦出力されます。
2. Drive上の画像ファイルが `Processed` フォルダへ移動されます。

## 4. 日常の運用への統合
出力されたタスクは、HMCの「朝の作戦会議（Morning Briefing）」や「Inbox Processing」などの通常のHMCデイリーワークフローを実行した際に、自動的にメインのタスクリスト (`tasks/backlog.md`) へ取り込まれます。

---

## トラブルシューティング
* **画像が読み込まれない:** フォルダIDが `.env` に正しく設定されているか、Inboxフォルダに画像が存在するか確認してください。
* **タスクが変な内容になる:** 手書き文字が極端に崩れている場合など、AIのOCRが誤認識することがあります。チャット画面（Step 3）で修正を指示してください。
