# HMC Task Management System ユーザーマニュアル

## 概要
HMCタスク管理システムは、**「Antigravity Agent（副操縦士）」** と **「Markdown（構造化データ）」**、そして **「Google Keep/Calendar（インターフェース）」** を組み合わせた運用オーケストレーションシステムです。
あなたの「副操縦士」として、日々のタスク整理、スケジュール調整、そしてHMC各アプリの実行を一元管理します。

## 主な機能・運用フロー

### 1. Morning Briefing（朝の作戦会議）
一日の始まりに、エージェントが「今日の行動計画」をまとめたレポートを作成します。

*   **コマンド**: 「ブリーフィング開始」「おはよう」など
*   **エージェントの処理**:
    1.  Google Calendarから今日の予定を取得。
    2.  `tasks/backlog.md` から期限到来タスクや定期タスクをピックアップ。
    3.  `tasks/active_tasks.md` の残タスクと統合。
    4.  **締め切り確認**: 期限が曖昧なタスクについて、実施要否やサポートの必要性を人間に問いかけます。
*   **アウトプット**: Google Keepに貼り付け可能なテキスト（スケジュール＋カテゴリ別タスクリスト）。

### 2. 日中の業務支援 (Vice Pilot)
チャットでの対話を通じて、タスクの操作やスケジュールの変更を行います。

*   **タスク追加**: 「〇〇をバックログに入れて」「今日のタスクに△△を追加」
*   **スケジュール調整**: 「明日の10時からMTGを入れて」 → Google Calendarに即時反映。
*   **アプリ実行**: 「請求書処理をやって」 → エージェントが裏側で `invoice_processor` 等を起動。

### 3. Daily Review（夜の完了報告）
一日の終わりに、完了した業務を報告し、ログとして記録します。

*   **コマンド**: 「完了報告」「今日はこれで終わり」
*   **処理**:
    1.  報告されたタスクを `tasks/active_tasks.md` で完了(`[x]`)にする。
    2.  完了タスクを `tasks/archive.md` に移動（アーカイブ）。
    3.  明日への申し送り事項があれば記録。

## ディレクトリ構成
タスクデータは全てテキストファイル（Markdown）で管理されており、直接編集も可能です。

*   `tasks/active_tasks.md`: 今日やるべきタスク（Today's Focus）。
*   `tasks/backlog.md`: 全てのタスク、定期タスク、いつかやるタスク。カテゴリ別に分類。
*   `tasks/archive.md`: 完了したタスクのログ。
*   `tasks/inbox/`: 一時的なメモや議事録。

## カテゴリ運用
タスクは以下のカテゴリで分類管理されます（Keepでの視認性向上のため）。
*   【年末年始宿題】
*   【開発】
*   【事務】
*   【連絡】
*   【家のこと】
*   ...など（必要に応じて増減可能）
