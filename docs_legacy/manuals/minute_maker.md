# Minute Maker (HMC 議事録メーカー) - ユーザーマニュアル

**Minute Maker** は、会議の文字起こしテキストから、人間との対話を通じて高精度な議事録を作成し、PDF化・アーカイブまでを一貫して行うAIエージェントスキルです。

## 概要と哲学
*   **Human-in-the-Loop:** AIが勝手に完成させるのではなく、重要な固有名詞、決定事項、コンテキスト（文脈）については必ず人間に確認を求めます。
*   **ワンストップ:** 文字起こしテキストの投入から、PDFの生成・保存までを一つのフローで完結させます。

## 利用方法（AIエージェント / Skill）

本機能は、AIエージェントのスキル `minute_maker` を使用して対話的に操作します。

### 1. 準備 (Input)
会議の文字起こしテキスト（`.txt`形式）を以下のディレクトリに配置してください。
*   `data/minutes_maker/input/`

※ ファイル名に日付や会議名を含めるとAIが自動認識しやすくなります（例: `20260210_運営会議.txt`）。

### 2. 実行開始
エージェントに以下のように話しかけてください。
> 「議事録を作って」
> 「minute_makerを起動して」

### 3. 対話フロー
エージェントが以下の順で質問・確認を行います。

1.  **話者特定:** `[話者ID] -> 実名` のマッピングを確認します。
2.  **テンプレート選択:** 会議の種類（定例、クライアントヒアリング、アイデア出し等）に応じたフォーマットを選択します。
3.  **コンテキスト補足:** アジェンダや重要キーワード（プロジェクト名等）を伝えます。
4.  **ドラフト確認:** 生成されたMarkdownのドラフトを確認します。

### 4. PDF化と保存
ドラフトの内容で問題なければ、「OK」「PDFにして」と伝えてください。
以下の一連の処理が自動実行されます。

1.  **PDF生成:** `weasyprint` を使用し、日本語フォント（Noto Sans CJK JP）を埋め込んだPDFを生成します。
2.  **アーカイブ:** 入力ファイル、ドラフト(MD)、完成PDFを `data/minutes_maker/meeting_files/YYYY/MM/` に移動・保存します。
    *   例: `data/minutes_maker/meeting_files/2026/02/20260210_運営会議.pdf`

## カスタマイズ
議事録のテンプレート（プロンプト）は以下で管理されています。新しい会議タイプを追加したい場合は、ここにMarkdownファイルを追加してください。
*   `.agent/skills/minute_maker/prompts/`

## トラブルシューティング
*   **文字化け:** PDFの日本語が豆腐（□□）になる場合、フォント設定に問題があります。`md_to_pdf.py` が正しいフォントパス（`data/minutes_maker/fonts/`）を参照しているか確認してください。
*   **ファイルが見つからない:** `input` フォルダに `.txt` ファイルがあるか確認してください。
