# Email Organizer User Manual

## 概要
Email Organizer は、Gmailの受信トレイを整理し、重要なタスクを抽出するためのスキルです。
AIとルールベースのハイブリッドアプローチで、「メルマガの一括アーカイブ」と「タスク（返信・確認）のリスト化」を行います。

## 主な機能
1.  **Learning Archiver（学習型アーカイブ）**:
    *   メルマガや通知を自動判定・アーカイブします。
    *   初回に「これはメルマガですか？」とユーザーに確認し、ルール（送信元・件名）を学習します。
2.  **Task Extractor（タスク抽出）**:
    *   重要なメールをAIが解析し、「要返信」「要確認」などのアクションと締め切りを提案します。
    *   ユーザーが承認すると、`tasks/mail_task.md` にToDoリストとして書き出します。

## 使用方法

### 1. 実行
プロジェクトの仮想環境 (`venv`) を使用してスクリプトを実行します。

```bash
# gogcliのパスワード設定が必要
export GOG_KEYRING_PASSWORD=your_password

# 実行
./venv/bin/python3 apps/email_organizer/organizer.py
```

### 2. インタラクティブ操作
スクリプトを実行すると、Inboxの状況が分析され、以下の3つのグループに分類されて表示されます。

*   **[1] Auto-Archive Candidates**: 設定に基づき自動アーカイブされるメール
*   **[2] Task Candidates**: AIが提案するタスク（要返信・要確認）
*   **[3] Review / Uncertain**: 判断できなかったメール

#### 操作コマンド
*   `y`: 表示されたプラン（[1]と[2]）を実行します。
*   `i`: インタラクティブモードに入り、[3]の不明なメールを1件ずつ確認します。
*   `q`: 何もせずに終了します。

#### インタラクティブモード (`i`)
不明なメールに対して、以下のアクションを選択できます。
*   `a`: **Archive & Learn**. メールをアーカイブし、その送信元を学習リスト (`config.json`) に追加します。次回から自動アーカイブされます。
*   `t`: **Task**. タスクとして追加（期限などの修正が可能）。
*   `v`: **Invoice**. **「請求書」としてマーク**。`Invoice_Pending` ラベルを付与し、受信トレイからアーカイブします（Invoice Processorで後日処理されます）。
*   `s`: **Skip**. 何もしない。

5.  **実行 (Execute)**:
    *   コマンド `y` を入力すると、確定したアーカイブ処理、タスク書き出し、請求書マークを実行します。

## 設定・データ
*   **学習データ**: `data/email_organizer/config.json`
    *   アーカイブ対象の送信元や件名キーワードが保存されます。誤って登録した場合はこのファイルを編集してください。
*   **出力**: `tasks/mail_task.md`
    *   抽出されたタスクが追記されます。

## トラブルシューティング
*   **gogcliエラー**: `GOG_KEYRING_PASSWORD` が正しいか確認してください。
*   **AI機能**: `.env` に `GOOGLE_API_KEY` が設定されていない場合、AI分析はスキップされ、簡易的なキーワード判定のみ動作します。
