## 3. expense_processor 初回リアル実走 → active 昇格(同 S37 後半)

ガクコ(Discord bot)から「経費フォルダを処理したいが expense SKILL が Garden に無い」と打診。

- **原因切り分け**: SKILL/service/cron は **実在**(S37 前半で配備済)。ガクコが「無い」と言うのは **bot.py の system prompt が daily-pilot 区画に閉じていて expense を案内していない**から(ファイルは claude が Read できる場所にある)。→ 移植は済んでおり、足りないのは「ガクコへの配線」だけと判明。
- **初回リアル実走(手動 A 経路 = Claude Code が VPS で実行)**:
  - extract: Drive 経費フォルダから **画像16 + CSV2本(PayPay / コスモ)= 77 行**抽出。`meisai202606.csv` は名前に反し Shift-JIS のコスモ明細だった(パーサは正しく判定)。
  - **77 件で `timeout 300` に到達**(画像 OCR 9 秒 × 16 + 61 行分類)→ timeout を外して再実行で完走。
  - ガクチョが **中間 CSV を直接レビュー**(VSCode)→ 個人利用分 18 行を除外 + 一部に部門付与 → 59 件 / ¥397,373。
  - dry-run(skip 0)→ 本登録。**57 件成功 / 2 件失敗**。失敗理由 = **会計年度期首日以前**(コスモ明細にレシート OCR の年誤読 2020・2023 が混入、他は全て 2026)。
  - 失敗 2 件を **2026 年に修正して再登録** → 全 59 件 Freee 反映。
- **ガクチョ依頼の改善を processor.py に実装**:
  - **partial 失敗 = 失敗行を `working/FAILED_{ts}.csv` に退避**(成功分はアーカイブして二重登録を防ぎ、失敗分だけ残す)
  - **exit code 1 + ログ末尾 `==NOTIFY==` アラーム**(cron / Discord ガクコ / 呼び出し AI が失敗を検知できる)
  - **全件失敗(登録 0)時はアーカイブ丸ごとスキップ**(再試行可能に)
  - **各行を try で包み、1 行の例外で全体を止めない**
  - dry-run で検証(不正費目 → exit 1 / 正常 → exit 0)
- **plot を draft → active に昇格**(cron + 手動は運用可。Discord「経費まわして」配線だけ残=改善余地)。

### 学び(初回実走)
- **合成テストでは出ない壁が実データで出る**:5 分タイムアウト / 会計年度期首日 / OCR 年誤読 / カード明細に個人分混在 / 不適切ファイル名(コスモなのに meisai)。本番1回が最良のテスト。
- **partial 失敗は「黙って一部落とす」が最悪**。退避 + exit + アラームで「何が登録されなかったか」を必ず残す。
