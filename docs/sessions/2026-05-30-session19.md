## 1. 朝チェーン初フル稼働の体感(口火 → 4 課題提起)

5/30 朝のチェーン(06:25 recurring-spawn / 06:30 morning-briefing / 06:40 morning_greet)は **無事完走**(Triage 2件・期限超過7件・本日1件)。だが届いた口火文面に対し、庭師から4課題提起:

1. **振り返り漏れ** — 前夜の active 上で書き換えた締切が backlog に未反映 → 翌朝「期限超過」として再出現
2. **表示** — タスクが `/` 区切りで横並び圧縮 → iPhone で読みにくい
3. **Triage 構成** — Q1/Q2/Q3(暫定締切 / 自然言語期限 / AI 支援)が **受け身**で芯を食わない。欲しいのは「過ごし方の提案」+「AI 支援提案」の能動 2 軸
4. **最後の一言** — 「予定と手持ち、どう組む?」が汎用文・意味なし

庭師の根本問い:「**HMC SKILL の動きは秘書のように先回り、ガクコは機械的読み上げ。systemprompt より SKILL ベースで書くべきか?**」(モデル独立性 / 運用工数の懸念込み)

## 2. 根本原因の特定 — HMC SKILL との比較

[HMC `.agent/skills/hmc_pilot/SKILL.md`](../../../harappa-cockpit/.agent/skills/hmc_pilot/SKILL.md) と HMG seed prompt を読み比べ:

| HMC SKILL の3層構造 | HMG seed への配置 |
|---|---|
| Core Philosophy(Human-in-the-Loop / SSOT / Pattern A / Empowerment & Proactivity) | **欠落** |
| Operational Modes(Mode 別 Step) | seed prompt に詰め込み |
| Output Style(Vice Pilot persona、Quick Wins/Must Do 強制) | **欠落** |

→ 「業務観」と「出力規範」を置く場所が HMG にない = 同じ Claude が読んでも **機械的読み上げ** になる。庭師の直感「SKILL ベースで書くべきか」は正しい。

## 4. 責務マップを書いて SKILL 化の合意

庭師の懸念「ガクコとコアが分離していくと、毎回どちらが何を担うかで混乱」「HMC は SKILL 1ファイルで運用できた軽さ」に応えるため、**現状の全要素を「層」と「責務」で整理した責務マップ**を提示(起動層 / 実行層 / 頭脳層 / 知識・指示層 / 状態層 / 入出力層)。

判明:
- HMG は HMC に比べ「経路」が増えた(cron × Discord × event)分、業務観を共有する場所が散らばっていた
- SKILL を **区画ごとに置けば** モデル独立・チャネル独立で全経路が同じ業務観を共有できる
- 境界: 「業務観 = SKILL」「人格 = persona」「チャネル特性 = Python」の3層に明示

→ 合意:**daily-pilot SKILL.md を書いて 5 経路を SKILL 参照型に振り替える**。

## 7. reschedule ライブ検証成功(S17/S18 持ち越し3セッション消化)

庭師が Discord で期限超過7件の締切再設定を bot に依頼 → **backlog + active 両方が同期反映**。bot は SKILL Mode 2 通りに動作:
- backlog の各タスクの `(MM/DD締切)` を新日付に書き換え
- active の同行も同期
- さらに **bot 独自の工夫**:active 冒頭に `## 運営・企画(繰越)` セクションを新設して reschedule した7件を集約 → 「今日締切」と「未来送り」が視覚的に分離(SKILL に明示してない動き、一過性なので副作用なし)
- 追加3件(伊東さんレス / 大阪請求 / パナソニックホームズ企画書作成)も対話で backlog/active に書き込み完了

→ S17 sandbox 通過後、S17/S18 と本番未確認だった reschedule 経路が **今日の対話の流れで自然にライブ検証成立**。短期1で入れた active→backlog 反映ロジック + SKILL ベース対話 + Mode 2 編集権限の三位一体が初めて動いた。

[docs/decisions/2026-05-30-skill-and-seed-separation.md](../decisions/2026-05-30-skill-and-seed-separation.md)

要点:
- 種 frontmatter:trigger / execute / outputs / pruning / on_failure 等(「いつ」「どんな入力で」)

- モデル独立・チャネル独立を SKILL が保証
