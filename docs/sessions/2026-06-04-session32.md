## 2. 道 A = 社内 LINE サーバ実装(中心成果)

ADR [2026-06-03 vendor-neutral-interaction-layer](../decisions/2026-06-03-vendor-neutral-interaction-layer.md) の実装順序「2」。

### 設計の要点
- **ベンダー中立の規律**:gaku-co5.0 は `switch.py`/`agent.py` に `import anthropic` が散在。**そのままコピーせず Provider アダプタに集約**(ADR 決定2)。gate/respond/app は `brain/provider.py` の `chat()` だけを呼ぶ。
- **記憶は Garden ネイティブ**:gaku-co5.0 の SQLite は移植せず、`memory_logger.append_turn("line_core_team", …)`(既に scope 引数対応)+ 直近 RAW ロード。三層分離に乗る。短期履歴はプロセス内リング(group 単位、再起動で消えるが gate には十分・永続は RAW が担う)。
- **MVP の tool は echo のみ**:本物の業務 tool は次フェーズ(測量士の手紙参照)。今は配線を通す。
- **エアギャップ**:app.py は設定された `LINE_CORE_TEAM_GROUP_ID` の **group のみ** 処理。社外グループ・個人 1:1・未知 source は構造的に無視 = スタッフ情報の漏洩経路を作らない。
- **情報境界**:respond は `capabilities.tools_for(scope)` で絞った tool だけ Provider に渡し、tool 実行前に再ガード(二重)。context は **core_team 自身の RAW のみ** load(master の私的 wiki/RAW を混ぜない)。

## 学び

- **「アダプタ集約しながら移植」は実際に効く規律**:gaku-co5.0 の async + `import anthropic` 散在を、同期 gate/respond(Provider 経由)+ FastAPI が executor で包む形に再構成。gate/respond は実 API なしでテストでき、LLM 差し替えはアダプタ 1 枚に局所化された。
- **エアギャップは "許可リスト" で実装すると堅い**:「社外を弾く」ではなく「core_team group **だけ** 通す」。未知 source はデフォルト無視。漏洩経路が構造的にゼロ。
- **情報境界は capability(行動)+ context(知識)の両輪**:tool を渡さない + 私的記憶を load しない、の二重で機微を遮断。prompt 頼みにしない。
