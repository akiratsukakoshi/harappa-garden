## (A) 早朝フェーズ — cron 実証と writeback 堅牢化

### 起きたこと

cron は予定どおり発火し、VPS 側では launcher が完走・MD も正しく更新された。**しかし Obsidian には届かなかった**。原因は 2 つ:

| # | 原因 | 修正 |
|---|---|---|
| 原因1 | Node の `fs.watch(recursive)` が時間経過で **既存ファイルの変更イベントを silently 取りこぼす**。新規作成は拾うが、起動数時間後の modify を落とす | `SCAN_INTERVAL_MS`(既定 15s)ごとの **mtime ベース reconcile scan を信頼できる backbone** に追加。`fs.watch` は低レイテンシ fast-path として残す(ベストエフォート) |
| 原因2 | 初回スキャンの burst が vault 全体(58 ファイル)を push。LiveSync の **Case-Sensitive = OFF** による `_id` 小文字化を未対応で、大文字 `_id` の **重複 doc を 14 件作成** | `_id = path.toLowerCase()`(`path` フィールドは元の大文字保持)+ スコープを `WRITEBACK_SCOPE`(`hmc_tasks/,garden/`)に限定。重複 14 件は `_bulk_docs` で `_deleted:true` 削除(原本の小文字 doc は無傷) |

### 検証

- 今朝のタスク(active_tasks rev 112)が CouchDB → Obsidian に到達、塚越さんが全タスクを確認。
- スコープ限定により対象が 16 ファイルに絞られ、全て小文字 ASCII パスで `_id` 衝突なし。
- 修正版の push 動作(scoped / lowercase `_id` / `h:+` chunk)をライブ確認。

## 達成感

Phase 3a は「実装が動いた」(S13)→「Obsidian に届いた」(S14)と段階を踏んできたが、**自律 cron が無人で回り、その成果が塚越さんの手元に届く** ところまでは実証されていなかった。

S15 早朝、まさに無人 cron が回った結果「VPS は更新されたのに Obsidian に届かない」という、S14 の手動検証では見えなかった故障モード(`fs.watch` の経年取りこぼし)が初めて表面化した。reconcile scan backbone を据えて修正し、今朝のタスクが Obsidian に到達 → **Phase 3a が「無人で end-to-end 動く」ことが初めて実証された**。

これをもって daily-pilot 3 本を active 化。Garden は「設計された種が、cron で自律的に起き、成果を庭師に届ける」生態系として、最初の 3 本が **正式稼働** に入った。
