## 0. セッションの起点 — 事前確認 + スコープ判断

→ 種 `monthly-shift-survey` を確認すると `target_month: $(date -d '+2 months' +%Y-%m)` = **8 月分** を製作する設定だった。ガクチョの想定(7 月)と 1 ヶ月ずれ。

ガクチョ判断 A(「6/1 発火 = 翌月(7月)分を出す」)で +1 month に修正。連動して関連ファイル(SKILL / services README / generate_shift_form.py docstring / month-end-working-hours-prep)の「翌々月」呼称も統一。

並行して recurring_master 月末項目を 2 つに分割(ID 規約に従い r010 リネーム + r016 新規):
- 旧: `シフトアンケート` (r010)
- 新: `シフトアンケート回答チェック` (r010) + `コドモンCSVを設置` (r016)

その後、本題のスコープを 3 案で提案 → **案 X(Stage 1 完成 + Stage A.5 は設計議論まで)** 採用。

## 3. 種 mycelium/index-refresh.md 起草

[garden/seeds/mycelium/index-refresh.md](../../garden/seeds/mycelium/index-refresh.md) 新規。要点:

- 毎日 03:00 JST(夜間バッチ枠)発火
- find で過去 24h 編集された soil/*.md を列挙(**index.md / log.md は自己ループ防止のため除外**)

- 自律実行(pruning.channel: none / post_approval: none)
