## やったこと

### 0. スコープの握り(壁打ち)
- generative は2サブドメイン = **見積・請求(構造的・決定論的)** と **提案書(creative・pptx)**。ガクチョ判断で **見積・請求を先に、提案書は別アプローチで後日**。
- **提案書の論点整理**: cowork(Anthropic 固有サービス)は ① ナレッジが soil に残らない ② ベンダーロックイン。Garden に移せば両方解消(モデルは Opus のまま=Claude Code セッション、md は soil 蓄積)。第1号サンプルは一丸ファルコス(docx 打合せ log → MD バージョン → PPTX)。
- ガクチョ確認「skill/tool は Claude に依存してない?」→ **三層で裏取り**:知識=md / 手順=SKILL(自前 picker+loader・標準 Skills 機構でない)/ 道具=service の Python(`anthropic` を import しない)。LLM は差し替え可能な実行者。

### 1. 過去 Excel からの型抽出
- 雛形の正本所在を特定(`請求書サンプル.xls` + 各受託フォルダの見積 .xls)。
- xlrd で MTI・三菱UFJ・パナHM の見積/請求を解析 → **3発見**:① 見積と請求は列レイアウトが別物(雛形2つ)② 研修・イベント運営型の標準明細が共通 ③ **見積に内部原価メモ**(粗利・利益率)が隠れている=クライアント非開示。
- → [soil/finance/templates/明細パターン.md](../../garden/soil/finance/templates/明細パターン.md)(研修・単発型 / 継続運営型 / 内部原価試算)。

### 2. 発行エンジン(issue_document.py)
- ガクチョが `請求書サンプル.xls` / 三菱UFJ見積を **.xlsx 保存**して提供 → service の雛形に。
- openpyxl で構造解析:数式生存(`D=B*C`・`SUM`)、差し込み先セルを特定。
- [issue_document.py](../../garden/services/finance/issue_document.py) = 構造正本 md(frontmatter)→ doc 種別で雛形選択 → 可変セルだけ差し込み(明細クリア→書き込み、数式は雛形のまま再計算)→ .xlsx 出力。**openpyxl のみ・LLM 非依存**。
- 実出力 GREEN:MTI 請求15行(税抜2,067,016)・三菱UFJ 見積6行(税抜900,000・税込990,000=実物一致)。

### 3. ガクチョ目視フィードバック → 仕上げ
- **和暦**: 日付セルに `[$-411]ggge"年"m"月"d"日"` 書式(値は日付のまま表示だけ和暦)。請求=請求日/振込期限・見積=見積日/有効期限。
- **件名(請求 A9)**: ボールド + 行高24。
- **★社印PNG消失 → 根本原因 = Pillow 不在**。openpyxl は Pillow が無いと画像を読めず save で落とす。Pillow を入れると雛形の社印を load→save で保持(位置も元通り col5/row11・col5/row9)。一旦作った社印再挿入ロジックは不要になり撤去(死コード掃除)。requirements に Pillow を必須化。
