## 学び・メモ

- **bot 再起動の PIDFILE ガード誤判定**(2回 bot 一時停止 → 復旧): run-bot.sh は `.bot.pid` を `kill -0` で見て生存判定。pkill 後の graceful shutdown(十数秒)中に呼ぶと「生存」と誤判定してスキップ。固定 sleep では不足 → **完全終了をポーリングで待つ**手順を memory 化([[bot-restart-procedure]])
- **report と SSOT**: ガクチョが「既存シートを使う」→「やはり新規作成」と方針変更。新規 ID を採用。HMC の SA(`credentials.json`)と Garden の SA は別認証なので、シート/フォルダは Garden SA に共有が必要
- **creative 区画は種プロンプトで Claude が中身を作る**: 財務系(expense/invoice)は processor が抽出 → Claude は承認だけだったが、SNS はセレクト・文案を Claude 自身が担う(画像 Read + 文体生成)。processor は下準備(DL・予約・レポート)に徹する
