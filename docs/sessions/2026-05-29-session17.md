# 2026-05-29 セッション17 — 朝の対話を立ち上げる(カレンダー移植 + 口火 + 会話書き戻し)

参加者: ガクチョ(塚越暁)、Claude (Opus 4.7)

> セッション16(garden-gaku-co 立ち上げ:夜のレポート + 喋るガクコ)の続き。
> 「朝のガクコ連絡が届かない」という気づきから、**朝の対話**を設計・実装したセッション。

## きっかけ — 「朝のガクコ連絡が届かなかった」

ガクチョが朝、Discord にガクコからの連絡が無いことに気づいた(Obsidian には active タスクは生成済み)。調査結果:

- **故障ではなく未実装**。S16 で作ったのは夜のレポート(22:40)と会話 bot だけ。**朝の Discord 配信機構が存在しなかった**。
- morning-briefing(06:30)は正常完走し active_tasks を生成 → Obsidian に出るが、生成した `==NOTIFY==` を push する朝版スクリプトが無かった。

## 開発フローの基本方針を確定(memory 記録)

ガクチョの要望:**実装に入る前にまず想定 UX を整理・確認し、当初計画＋高解像度な UX イメージで方向を確定してから実装を固める**。フェーズが進むと解像度が上がるため、当初計画を絶対視せず上書き・具体化していく。→ 全プロジェクト共通の memory に記録(`ux-first-dev-flow`)。

## 確定した朝の対話 UX

夜レポートの朝版(一方向)は**不要**。朝は**対話型**にする。

| 論点 | 決定 |
|---|---|
| 口火 | **gaku-co から声かけ**(報告ではなく対話の口火)。06:40 に投稿して待つ |
| Triage 提示 | **まとめて提示**(全項目を一度に、自由返信) |
| 書き戻し | **即書き＋軽い報告**(master = ガクチョ本人、Obsidian で後修正可) |
| 確定 | Triage 全消化で「確定でいい?」と一度確認。**自動確定はしない** |
| #1 | triage を active_tasks.md の**最下段にもミラー**(1画面で概要が見える) |
| #2 | 口火に **active_tasks の内容**を載せる |
| #3 | カレンダーを見て予定と手持ちを整理(HMC のカレンダー認証を移植) |

## 実装(提示順に #3 → #1 → #2 → #3/#4)

### カレンダー認証の移植 — [garden/services/calendar/](../../garden/services/calendar/)

- HMC(`harappa-cockpit`)の `manage_calendar.py` を Garden に移植。**MCP を立てずに**解決(MAP 宿題を移植で消化)。
- **クリーンな Production トークンを再同意で発行**(Testing 由来の7日クロックを回避)。access token は常に約1h・refresh token は Production で無期限、という2種類の expiry を整理。
- VPS に認証情報を配置(chmod 600・git 除外)、google ライブラリを garden-gaku-co venv に共用導入、実データ取得を確認。
- morning-briefing への配線 = **launcher の `computed_inputs` で事前取得 → prompt に注入**(claude に credential/MCP を渡さない)。launcher の shell timeout を 5s→15s に。

### #1 — triage を active_tasks 最下段にミラー

morning-briefing 種の prompt に `## 🔖 Triage(対話で消化 / 詳細は board)` 最下段セクションを追加(簡潔な一覧、詳細 Q は board のまま)。

### #2 — 朝の口火 — [garden-gaku-co/morning_greet.py](../../garden/services/garden-gaku-co/morning_greet.py)

- active_tasks.md を**そのまま要約転記**(二重管理を避ける)。挨拶・締めは固定文(claude 不要=高速確実)。
- ヘッダ日付が今日でなければ警告(朝のヘルスチェック兼用)。
- cron **06:40**(06:30 ブリーフの完走を待つバッファ)。サンプルデータで test 送信して文面確認 → OK。

### #3/#4 — 会話の書き戻し(read-only 解除)— [garden-gaku-co/bot.py](../../garden/services/garden-gaku-co/bot.py)

- bot の claude に Read/Edit/Write を解禁(settings.json で `garden-mirror/{hmc_tasks,garden}` に path-scoped allow 済。Bash 等は禁止)。
- 反映ルール: 完了 `[ ]`→`[x]` / 締切変更は backlog(正本)/ 追加は `## 追加` / Triage 回答は board / `## スケジュール` は触らない。**即書き＋軽い報告**、全消化で「確定でいい?」。
- **sandbox(本番データ非接触)で実編集を検証** → 完了・追加・Triage 両回答すべて正しく編集、backlog は無編集(締切変更指示なしのため正)、status は確認待ちで保持。
- ライブ bot を再起動して本番投入 → ガクチョが Discord で実地確認、問題なし。
