# セッション59(2026-06-25)— VPS cron 全停止を根治:`claude -p` 認証を長期トークンに切替

## 発端

ガクチョ「昨晩の夜のレビュー以降、cron エラーが続いている。確認をしてシューティングをお願い」。
番人(Watcher)が Discord master に 6/24 22:40 以降 **5 件のアラート**を連投していた:

| 時刻(JST) | 種 | 内容 |
|---|---|---|
| 06/24 22:40 | daily-pilot/night-review | exit=1 |
| 06/25 03:10 | mycelium/index-refresh | exit=1 |
| 06/25 03:40 | mycelium/ingest-raw | 401 / exit=1 |
| 06/25 04:00 | mycelium/consolidate-wiki | 401 / exit=1 |
| 06/25 06:30 | daily-pilot/recurring-spawn | exit=1 |
| 06/25 06:40 | daily-pilot/morning-briefing | exit=1 + active_tasks 日付ズレ警告 |

## 診断 — 単一の根本原因

全種のログ末尾が同一:
```
---- claude -p output ----
Failed to authenticate. API Error: 401 Invalid authentication credentials
exit_code: 1
```

VPS の Claude 認証は session9 で `claude login`(個人サブスク OAuth)で設定されていた。
`~/.claude/.credentials.json` を調べると(秘匿値は出さず期限フィールドのみ):

- `expiresAt` = **2026-06-24T05:25 UTC(=14:25 JST)に失効**、subscriptionType=`pro`
- 同時刻、ローカル端末は subscriptionType=`max` で新しく有効(expires 6/25 06:12 UTC)
- VPS で `claude -p` を叩くと今も即 401 → アクセストークン失効 + **リフレッシュトークンも拒否**

**真因の構造**:同一アカウントをローカルで再ログイン(Max 化)した際に
**OAuth リフレッシュトークンが回転し、VPS 側セッションが無効化**された。
個人サブスク `claude login` をヘッドレス cron で回す構成の構造的弱点。
朝の「active_tasks が今日でない」警告も、night-review が落ちて active を更新できなかった派生症状。

## 対応 — 恒久(`setup-token` + cron-secrets.env 配線)

`claude login`(対話セッションと回転衝突する)をやめ、**`claude setup-token`** で
**1 年有効・ヘッドレス専用の長期トークン**を発行(対話ログインと回転衝突しない)。

1. ガクチョが VPS で `claude setup-token` 実行 → ブラウザ承認(Max アカウント)→ `sk-ant-oat01-...` 発行
2. トークンを VPS `~/.claude/cron-secrets.env`(chmod 600, `CLAUDE_CODE_OAUTH_TOKEN=...`)に配置。**値は repo に置かない**
   - 初回、接頭辞 `CLAUDE_CODE_OAUTH_TOKEN=` が抜けて bare token になっていたのを、値を表示せず付け直して修正
3. **launcher.mjs(repo 正本)に cron-secrets ローダー追加** — 起動時にこのファイルを読み `process.env` に注入。
   spawn 時 `{...process.env}` で `claude -p` に継承される(cron は最小 env なのでこの配線が必須)。`GARDEN_CRON_SECRETS` で上書き可
4. repo launcher を VPS にデプロイ(構文チェック GREEN)。**副次効果 = VPS は repo HEAD より古い launcher を動かしていた drift があり**、
   今回のデプロイで S54 の MCP フラグ対応も VPS に追従して解消

検証:`set -a; . ~/.claude/cron-secrets.env; set +a; claude -p 'reply OK'` → `AUTH_OK`(値は非表示で使用のみ)。

## 復旧 — 落ちた種の手動リカバリ(全緑)

launcher 経由(= cron と同じ経路)で再実行し、env 読み込みが効くことも end-to-end で確認:

| 種 | 結果 |
|---|---|
| daily-pilot/recurring-spawn | ✅ exit 0・当日定期2件を backlog へ |
| daily-pilot/morning-briefing | ✅ 6/25 ブリーフ生成・active_tasks `2026/06/25 (木)` 更新・Triage 6件 |
| mycelium/index-refresh | ✅ [ok] |
| mycelium/ingest-raw | ✅ [ok] |
| mycelium/consolidate-wiki | ✅ [ok] |
| daily-pilot/night-review(6/24分) | ⏭️ **あえて再実行せず** |

**night-review を再実行しなかった理由**:launcher は実行時刻から日付変数を計算するため、
日中に走らせると「6/25 の夜レビュー」扱いになり当日タスクを誤って締めてしまう。
6/25 分は今夜 22:30 の定刻 cron が正常処理する。
