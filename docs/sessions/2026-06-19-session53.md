### 2. 新区画 scribe(会議録の番人)を新植(plot_gardener seedling)
ガクチョの課題:**Plaud は録音を汎用タイトルで自動命名 → 手作業でクライアント名を付与 → 漏れると soil に取り込まれない**。

**相談で確定した設計**:
- **Plaud MCP/API は read-only**(list/get/note/transcript のみ。rename・フォルダ作成/移動・metadata 編集はどこにも未公開)。Embedded/Developer API(OAuth・portal.plaud.ai)は「音声を投げて文字起こし」用の**白ラベル別名前空間**で、ガクチョの既存録音には触れない。→ **自動リネーム/フォルダ振り分けは断念**(ガクチョ判断)。
- **Plaud MCP は OAuth 対話前提でヘッドレス cron(VPS)から到達不可**(client_steward が積んだ「MCP ブリッジ」homework と同根)。
- → **ハイブリッド設計**:(A) **漏れ防止** = Garden が**タイトル非依存**で取り込み(カレンダー×内容×履歴で主体判定 → soil 振り分け)/ (B) **タイトル整備** = 正規タイトルを**提案**、ガクチョが Plaud アプリで手動リネーム(整ったタイトル = 取り込み済みの目印)。
- **タイトル表記ルール** = `{月日} 【{主体}】会議タイトル`(年なし・社外社内ラベルなし・主体=クライアント名 or 社内会議体 or メンタリング相手 or イベント名、個人は主体省略)。
- **情報セキュリティ(3層モデル・S53 確定)**:Tier1 メタ + Tier2 要約(**具体数字を含む**)= soil 常駐 / Tier3 生 transcript・生 AI サマリ = soil に置かず MCP で都度。`get_note` の生出力は貼らず**要約し直す**(生 dump 禁止)。**数字は消さない**(可読性 + finance 突合のため)。機密対策は「生 transcript を外す + private repo + `confidential`」で担保。→ [scribe SKILL](../../garden/plots/scribe/SKILL.md) Step 3 + [soil/clients/README](../../garden/soil/clients/README.md) に明記。
