## 1. A: memory 正本ルール ADR + memory-sync スクリプト 2 本

### 1.1 設計の出発点 — 現状把握

| 場所 | 状態 |
|---|---|
| repo `garden/memory/master/{raw,wiki}/` | 構造ファイル + wiki 3 件(README, daily_operation, tech_infra, index) |
| VPS `garden-mirror/garden/memory/master/{raw,wiki}/` | 同構造 + raw 3 件(2026-05-31〜2026-06-02)+ wiki 同 3 件(うち tech_infra は 03:34 で repo より新しい) |
| VPS `/home/vps-harappa/garden/memory/` | **存在しない**(board/log と違って memory は vault 外に出していない) |

重要な発見:
- memory wiki は **VPS で 03:30 ingest-raw が毎晩積み上げる** = soil の log.md と同じく「VPS で書き込みが続く構造ファイル」
- raw は **bot.py 専属で `.gitignore` 除外済**(S22 ADR 通り)
- 物理配置・正本所在・sync 経路が **明文化されていなかった**(S26 で soil は整理済、memory は持ち越し)

### 1.2 正本ルール(ファイル種別ごとに分離)

| パス | 書き手 | 正本 | git | sync |
|---|---|---|---|---|
| `memory/README.md` | 人(Claude) | repo | ✅ | pull/push |
| `memory/master/raw/.gitkeep` | 人 | repo | ✅ | pull/push |
| **`memory/master/raw/*.md`** | bot.py / memory_logger.py | **VPS 専属** | ❌(.gitignore) | **除外** |
| `memory/master/wiki/*.md` | ingest-raw 種(VPS) + 人(repo / vault) | **VPS 主・repo 従** | ✅ | pull/push |
| `memory/master/wiki/index.md` | 同上 | 同上 | ✅ | pull/push |

「VPS 主・repo 従」の意味: ingest-raw が毎晩 03:30 に積み上げる主流は VPS。repo は (1) git 履歴を残す (2) Claude が手で編集して push する経路。**セッション開始時 pull → 編集 → 終了時 push** の規律で競合回避。

### 1.4 検証

```
pull dry-run → README.md + master/{raw,wiki}/ 構造のみ rsync 対象
              raw/*.md は除外(2026-05-31〜06-02.md が rsync 一覧に出ない)✅

push dry-run → README.md + .gitkeep のみ push 対象 ✅
push 実走   → VPS に README 反映
```

### 1.5 採用しなかった案

ADR に詳述。要点だけ:

- **案 A: wiki も VPS 専属**(repo に上げない) → Claude が wiki を編集する場面が確実にある + git 履歴を残したい → 却下
- **案 B: repo を完全な正本**(VPS は実行用キャッシュ) → ingest-raw が VPS で毎晩書く頻度 >>> Claude の編集頻度 → 設計と現実をずらすメリットなし → 却下
- **案 C: 双方向同期(unison 等)** → 仕組みが重い + 競合自動解決はブラックボックス化 → 却下
- **soil-sync と 1 本に統合(`garden-sync`)** → soil と memory は対象も除外ルールも違う + どちらかだけ実行するケースあり → 別ディレクトリで責務分離
