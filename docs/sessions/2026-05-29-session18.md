## 1. COUCHDB_PASS の rotation(主題・最優先債の返済)

S14 で writeback-daemon デバッグ中、`wget --password=...` のエラー出力に CouchDB admin パスワードが露出 → AI 会話トランスクリプトに残存。S15〜S17 にわたり「最優先」宿題として3セッション持ち越されていたものを返済。

### 把握したアーキテクチャ

- **単一の共有 admin パスワード**(`garden-admin`)が **5消費者**で使い回し: CouchDB 本体 / mirror-daemon(`.env COUCHDB_PASS`)/ writeback-daemon(同)/ Obsidian LiveSync = PC・iPhone(basic auth)。
- **E2EE パスフレーズは別物**で露出も変更もなし(暗号化データはそのまま復号可)。
- git には `.env.example`(placeholder)しか追跡なし → **git 露出なし**。garden-gaku-co / launcher は CouchDB 認証を持たない(MD ミラー経由)ので対象外。

### 実施(全て VPS 上、値はチャットに一切出さず)

1. 新パス生成(`openssl rand -base64 48 | tr -d '\n/+=' | head -c 32` = 英数字32文字)。
2. CouchDB admin を **`_config` API** で変更(旧パスで認証)。即時反映 + **host マウント `local.ini`(config チェーン最後)に永続** = コンテナ recreate でも生存することを running hash == host hash で確認。CouchDB 再起動不要。
3. 3つの `.env`(couchdb=`COUCHDB_PASSWORD`、daemon×2=`COUCHDB_PASS`)を新パスへ書換。
4. daemon 2本を recreate(`--force-recreate` は固定 `container_name` と衝突 → **`stop && rm && up`**。state は host ボリュームに残存)。両者 401 なしで再接続(mirror=`_changes`購読→put、writeback=reconcile→skip比較)。
5. Obsidian は新パスを VPS の `~/garden-newpass.txt`(chmod 600)に置き、**ガクチョ自身の別ターミナルで `cat`**(AI チャット非経由)。PC は Password 欄のみ更新、iPhone は Setup URI で再構成 → 両端末「サクセス」。
6. 受渡ファイル shred / VPS bash 履歴に CouchDB パス残存なしを確認。

### 詰まった点と知見

- `_config` PUT 直後の検証で **新パスも一瞬 401**(伝播ブリップ)→ 数秒後には正常。慌てて二重変更しないこと。
- `/_up` も認証必須(`require_valid_user = true`)。
- この VPS は **docker-compose v1**(ハイフン)。

## 2. 旧 n8n の撤去(rotation の履歴掃除で発見 → 派生)

bash 履歴掃除中、別 secret として旧 n8n の `N8N_BASIC_AUTH_PASSWORD` 平文が発覚。ガクチョ「もう使っていない、関連ファイルがあれば削除して」。

- docker 側は既にクリーン(コンテナ/イメージ/ボリューム/ネットワークすべて無し)。
- `~/.n8n`(166M)+ `~/.n8n_backup_1.81`(154M)を、sqlite×複数(**認証情報**)/ `config`(暗号化キー)/ `ssh/` 鍵を **shred** 後に `rm -rf`。約320M 解放。
- bash 履歴の n8n 関連 36行を scrub(一時バックアップ → 検証 → shred)。
- **残**: NPM の proxy host #1 `n8n-harappa.duckdns.org`(upstream 消失=502 のダングリング)。手で `.conf` を消すと NPM が DB から再生成するため **NPM UI から削除が正道** → ガクチョ手元作業。
