## 1. 費目分類のバッチ化(タイムアウト対策)

S37 で 77 件(画像16 + CSV61行)が 5 分タイムアウトに当たった件の恒久対処。[processor.py](../../garden/services/expense-processor/processor.py)

- **CSV 明細の費目分類をバッチ化**: 1 行 1 Gemini 呼び出し → 複数行を 1 プロンプトにまとめ、JSON 配列で受けて
  **入力順に復元**(`ExpenseClassifier.classify_batch` / `_classify_chunk`)。`EXPENSE_CLASSIFY_BATCH_SIZE`(既定40)。
  61行=61往復が ~2往復に。
- **画像 OCR を並列化**: 画像はバッチ不可なので `ThreadPoolExecutor`(`EXPENSE_IMAGE_OCR_WORKERS` 既定4)。
  ファイル毎の `time.sleep(2)` は撤去(429 バックオフは各リトライに残存)。
- **安全装置**: バッチ失敗 / 行数不一致 / JSON 不正なら **その範囲だけ 1 行ずつフォールバック**(黙って全部
  消耗品費 にしない)。
- 検証: モック6ケース(順不同→順序復元・リスト外科目の部分一致吸収・欠落/不正→フォールバック・空・分割)全通過。

- **読み取り専用 `taxes` で実地検証 → 成功**(claude が scoped Bash で processor.py を実行し Freee の実コード取得)。

### ⚠️ permission matcher の gotcha(重要・再利用知識)

Claude Code の Bash 権限照合は **`Bash(<絶対パス接頭辞>:*)` 形式 + 単一コマンド**でないとマッチしない。

- 当初 settings.json にあった `Bash(.../.venv/bin/python *)`(末尾スペース星)や
  `Bash(cd ... && .venv/bin/python *)`(複合コマンド・相対パス)は **拒否される**(`&&` で分割照合され、
  相対 `.venv/bin/python` も絶対パターンに一致しない)。
- 正: `Bash(/home/vps-harappa/garden/services/expense-processor/.venv/bin/python:*)`。
  呼ぶ側も **絶対パス + cd なし**(processor.py は内部パスが絶対なので cwd 不問)。
- → bot.py / SKILL Mode 2 Step1 / 種 Step1 / board の `execute_command` を **すべて絶対パス形式**に統一。
- 副次発見: 既存の **shift-manager の Bash allow も旧 ` *` 形式**で未機能の疑い。ただし shift の実行は
  send_pending の subprocess / 手動経由なので実害は無さそう(要・別途確認)。

→ ガクチョの Discord 実テストで **「経費まわして」一連が完璧に機能**(やり取りも問題なし)を確認。

## 3. 件数が多い月の直接編集 → Google Sheets(案A)

ガクチョ要望: 件数が少なければチャットのやり取りでよいが、多い月は **直接編集したい**。Discord で回す前提
(スマホ可)で、案A(Google Sheets 経由)を選択。方式は shift_manager と同じ
**「ガクチョ所有ワークブックを SA に共有 → 毎月新規タブ」**(SA 所有ファイルがガクチョの Drive に出ない問題を回避)。

- [lib/sheets_client.py](../../garden/services/expense-processor/lib/sheets_client.py)(新規): gspread + 同 SA。
  `write_tab`(`{YYYYMM}` タブ作成・費目プルダウン5分類・要確認行黄色・ヘッダ固定・列は CSV と 1:1 位置固定)/
  `read_tab`(読み戻し。**金額が空/0 の行=削除扱いでスキップ**)。
- processor.py に **`to-sheet <csv>`**(→ `REVIEW_SHEET_URL` / `REVIEW_TAB` 出力)/ **`from-sheet <tab>`**
  (→ `REVIEWED_CSV` 出力)。
- フロー更新(bot / SKILL Mode 2・3 / 種): extract → **to-sheet** → board(`review_sheet_url`/`review_tab`)→
  Sheet URL つき通知。承認 → **from-sheet** → dry-run → 本登録。少量はチャット修正も可(Sheet を出した後は
  from-sheet が正 = 二重編集回避)。
- secret: `EXPENSE_REVIEW_SHEET_ID`(ガクチョが1回作成 → SA `harappa-drive-bot@harappa-cockpit.iam.gserviceaccount.com`
  に Editor 共有 → ID を .env)。gspread を venv 追加。
- **検証**: 合成6行で to-sheet → from-sheet ラウンドトリップ GREEN(要確認行保持・部門保持・列マッピング正・
  amount=0 行スキップで 6→5 行)。テストタブ掃除済み。
