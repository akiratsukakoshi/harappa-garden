### (4) で新たに浮上した壁 — mirror-daemon 権限

permission mode 検証実走で **`EACCES`** が判明:

- daemon コンテナが **root(uid 0)** で動作 → mirror に書き出すファイルすべて root 所有
- vps-harappa(uid 1000)で動く `claude -p` から書き込み不可

**修正**:
1. `docker-compose.yml` に `user: "1000:1000"` 追加(HMG repo 側にも反映済 → 次回 git commit へ)
2. `docker-compose -p garden-mirror-daemon down`
3. 塚越さん VPS で `sudo chown -R vps-harappa:vps-harappa /home/vps-harappa/garden-mirror /home/vps-harappa/garden/services/mirror-daemon/data`
4. `docker-compose -p garden-mirror-daemon up -d`

→ コンテナが node(uid 1000)で起動、last_seq から再開、書き出しが vps-harappa 所有になる。
