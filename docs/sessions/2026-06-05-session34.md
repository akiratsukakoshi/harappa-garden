## 越えた壁(重要事項)

### コンテナ化への方針転換(本セッション最大の判断)
- NPM はコンテナ。コンテナ内 `127.0.0.1` はホストの 8011 に届かない。
- ホストを docker gateway(172.20.0.1)で指しても、**VPS の iptables が docker→ホストを DROP**
  (8011 も NPM 自身の 81 も timeout で確認)。sudo 無しで firewall 変更不可。
- → LINE サーバを `proxy-manager_default` 上のコンテナ `garden-gaku-core` にして、NPM からコンテナ名で
  到達(couchdb / gaku-co5 と同方式)。`Dockerfile.line` + `run-line-container.sh`(docker run。
  compose v1 は buildkit イメージの `ContainerConfig` 欠落バグで使用不可)。ADR 化。

### NPM ログイン不能 → bcrypt 安全リセット
- email=`tukapontas@gmail.com`(唯一の admin)。パケットフィルタで管理画面に入れず、
  **SSH トンネル**(`ssh -fN -L 8181:127.0.0.1:81 harappa`)で UI を開通。
- パスワード不明 → ADR 2026-05-25 決定8 の手順(DB バックアップ → bcrypt 生成 → 60 字 assertion →
  `journal_mode=MEMORY` で UPDATE → round-trip 検証 true)で一時パスワード発行 → ガクチョが再設定。
- ハマり: bcrypt ハッシュの `$` がシェルで再展開され空 UPDATE になりかけた → quoted heredoc で回避。

### DNS は Xserver(Vercel でなく)
- 権威 NS = `ns*.xvps.ne.jp` = Xserver VPS。`harappa.monster` の DNS は Xserver パネルで管理。

### tool-use 400 バグ修正(provider / respond)
- respond の tool-use ループが assistant ターンを「テキストのみ」で積み、モデルの `tool_use` ブロックを
  欠落 → 続く `tool_result` に対応 `tool_use` 無しで Anthropic 400。
- 中立 Provider 形式に「assistant の tool_calls」表現を追加し、`_to_anthropic_message` で
  text + tool_use ブロック列に再構成。round-trip ペアリング検証 OK。
- placeholder の `echo` を 1:1 では渡さない(`offer_tools=False`)方針に(渡すとモデルが無駄に呼んで
  最終テキストが空 → reply が飛ばない事象)。

### テスト用エアギャップ例外

- ガクチョ userId = `U5816…6565` を登録 → 1:1 会話 GREEN。

## 学び
- **「ホスト process か container か」は環境の制約(ここでは firewall)が決める**。設計図(DEPLOY 初版)を
  鵜呑みにせず、コンテナ→ホスト到達性を実地に当てて初めて正しい配線が分かった。
- **secret を扱う VPS 操作は quoted heredoc + 三段(生成→assertion→反映)が効く**。bcrypt の `$` 再展開事故も、
  DB バックアップ + round-trip 検証で安全に着地。
- **placeholder ツールは会話品質を下げる**。配線確認用の echo を本番会話に出すと、モデルが無駄に使って
  空応答になる。tool は「本物ができてから」渡すのが正しい(plot_gardener の水面下方針と一致)。
