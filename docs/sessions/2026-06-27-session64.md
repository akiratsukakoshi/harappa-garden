## 設計判断

`plot_gardener` の hybrid として `meeting_coordinator` 区画を新設した。

`field_assistant` は現場準備リマインドの区画であり、会議調整・Zoom/Calendar発行・会議履歴入口とは責務が異なるため、別区画にした。ただし将来、`field_assistant` の企画MTG確認は `meeting_coordinator` へ移管する。

MVP は月次運営会議。

- 発火: 毎月第3月曜 08:30
- 対象: 翌月5〜9日の平日
- 候補: ガクチョ Google Calendar の空き時間
- 優先: 午前、午後、夜の順
- 標準尺: 90分。指示があれば60分等へ上書き可
- 他参加者の空き: LINE返信ベース
- 確定: ガクチョの明示指示
- 確定後: HARAPPA会社ZoomアカウントでURL発行、ガクチョCalendar登録、参加者email招待、core_team LINEへZoom URL通知

toC soil 連携は、MVPでは `meeting_type` / 参加者 / calendar event / zoom を state に残すところまで。将来、Plaud(scribe)、STORES予約、シフトカレンダー、Notionフィールドレポートへ接続する。

## 7月運営会議の本番テスト

7月運営会議は既に調整が進んでいたため、イレギュラーに候補2本で state を作成した。

- A: 2026-07-06(月) 08:30-10:00
- B: 2026-07-07(火) 08:30-10:00

core_team LINE へ `@少佐 @ゆーじさん` のメンション付きで候補を送信。発信はガクチョ確認でOK。

その後、ガクチョが「A案で運営会議を確定。zoom発行...」とLINE返信したが、ガクコから応答なし。調査結果:

- LINE webhook は受信済み
- gate は `should=True` / `dispatch=tools`
- ただし LLM が `confirm_meeting_coordination` tool を呼ばず、state は `open` のまま

対応:

- LINE webhook 側に決定的ルートを追加
- `運営会議 Aで確定` / `Aで確定` などを、LLM任せにせず `confirm_meeting` へ直結
- `meeting_id` なしでも最新の open `operations_monthly` を解決

最終的に CLI から A案を本番確定:

- Zoom URL 発行成功
- Google Calendar 登録成功
- 参加者 email 招待送信
- LINE 確定通知送信
- state は `confirmed A`

ガクチョ側でも LINE返信・Calendar掲載を確認済み。

## 運用調整

人間向けの文面から `meeting_id` を削除した。情報量が増えると読まれないため。

確定の定型:

```text
運営会議 Aで確定
```

調整依頼文には返信例を明記する:

```text
返信例:
・少佐/ゆーじさん: AならOK、Bは難しい
・ガクチョ確定: 運営会議 Aで確定
```

`A` だけの返信は確定扱いしない。参加可否か確定か曖昧なため。

少佐/ゆーじさん等の返信は確定ではなく、可否記録として扱う。ガクコは「可否を記録しました。ガクチョの確定判断を待ちます。」側に寄せる。

確定後のLINE通知は Zoom URL のみ。Calendarリンクは出さない(email招待で届くため)。

## secret / VPS

Zoom は Server-to-Server OAuth app を作成。

必要なenv:

- `ZOOM_ACCOUNT_ID`
- `ZOOM_CLIENT_ID`
- `ZOOM_CLIENT_SECRET`
- `ZOOM_USER_ID=me`

scope:

- `meeting:write:meeting:admin`

配置:

- `/home/vps-harappa/garden/services/meeting-coordinator/.env`

LINEは既存 `garden-gaku-co/.env` から、値を表示せず転記。

`python3-venv` がVPSに無く専用 `.venv` は作れなかったため、当面は既存の `garden-gaku-co/venv` を共有する。将来は `/home/vps-harappa/garden/runtime/venv` のような Garden共通 runtime venv に寄せるのがよい。
