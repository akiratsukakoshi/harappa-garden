# セッション57(2026-06-23)

## 一行サマリ

SNS 投稿の状況確認から始まり、**私(Claude)が「コードの著作元(local repo)」と「実行ランタイム(VPS)」を取り違えて基本構造を誤読**した。ガクチョの指摘で原因を追い、**board が「ローカル repo にも VPS にも存在しうる」二重構造になっていたこと**が根本原因と判明 → **board の住所を VPS 一箇所に統一**(scribe は outbox 経由・push 後に掃除・`--ignore-existing` バグ撤去)。あわせて **SNS の役割境界**(パイプライン/設定は私が整備・gaku-co は実行のみ)を確立、SNS 予約の権限欠落を私が修正 → gaku-co が新文案で予約完走。最後に lint ERROR(紛れ込んだ README)と WARN(`scheduled` 未登録)も根治。

## 起きた事象(時系列)

1. **発端**: ガクコより「SNS 投稿案が Discord で届いた。board を書き換え修正依頼したが、processor.py の `--caption-file` を `sns-manager/temp/` に書けない(権限)。a) update-config で恒久解決 / b) Claude Code UI で許可、どちらが良いか(a 推奨)」とガクチョに相談が来た。

2. **私の誤読①(構造把握ミス)**: 私は local repo の `garden/board/pending/` を見て「SNS board が無い・temp が空・何も予約されていない」と断定。実際は **SNS 区画は `execution_host: vps`** で、board・選定画像・予約はすべて **VPS 上**にあった(local を見て runtime を語った)。

3. **私の誤読②(役割境界)**: 「gaku-co が update-config で自分の権限を足す(a 案)」を一度推奨してしまった。ガクチョ是正 = **プログラム/設定/権限を直すのは Claude の仕事。gaku-co は完成したパイプラインを指示通り実行するだけ**。

4. **ガクチョの核心の問い**: 「memory で上塗りするな。**プロジェクト構造そのものが誤読させたのでは**? board の正本が VPS にしか無いと読み取れない構造になっていないか」。
   → 調査で **board が二重に存在する構造**を確認:
   - scribe(`execution_host: local`・Plaud がローカル専用)は board を local `garden/board/pending/` に書き、`run-local.sh` が VPS へ push。
   - VPS 種(SNS 等)は VPS に直接書く。
   - → `ls garden/board/pending/`(最も自然な操作)が**警告なしに不完全な答え**を返す。memory なしでは誰でも誤る。

5. **さらに実害2件を確認**:
   - **墓場化**: scribe が local board を push 後も掃除せず溜める(誤読の直接原因だった recording-sweep がこれ)。
   - **VPS が stale**: `rsync --ignore-existing` のため scribe の更新が VPS に届かない。実測で local(23:15・録音C=開発MTGの解析入り完全版)vs VPS(22:40・「判定不可」の古い版)が食い違っていた。

6. **lint ERROR**: ガクコから「README.md: from_seed が無い」。原因 = 私が**先の(撤回した)アプローチで作ったガード README** が、自動 poll の旧 `run-local.sh`(`rsync *.md`)で VPS board/pending に紛れ込んでいた。

7. **発見(良いニュース)**: 上記対応中に、**gaku-co が既に正しい新文案で SNS 予約を完了**していたと判明(temp の caption ファイルで検証)。

## 対応(やったこと)

### A. board の住所を VPS 一箇所に統一(誤読①の根治・構造修正)

| 対象 | 変更 |
|---|---|
| [scribe 種](../../garden/seeds/scribe/daily-recording-sweep.md) | board 書き出し先を `board/pending` → **`garden/services/scribe/outbox`**。新規録音がある時だけ書く。冪等は processed.jsonl。ローカル notified_at チェック廃止 |
| [run-local.sh](../../garden/services/scribe/run-local.sh) | outbox → VPS へ push、**`--ignore-existing` 廃止**(更新が届く)、**push 成功後 outbox を空に**(墓場を作らない) |
| local `garden/board/pending/` | **ディレクトリごと撤去**。repo の `garden/board/` はルール正本(README)だけ |
| [.gitignore](../../.gitignore) | board/pending の前提を消し `scribe/outbox/` を ignore |
| [board/README.md](../../garden/board/README.md) | 「board の住所は VPS 一箇所・repo に board コンテンツ無し」+ S57 の理由を明記 |
| ADR | [2026-06-23-board-single-home-on-vps.md](../decisions/2026-06-23-board-single-home-on-vps.md) |

実害是正: `--ignore-existing` で古かった VPS の recording-sweep を、ローカルの完全版で上書き(notified_at 保持=再通知なし)。

### B. SNS 役割境界(誤読②の根治)+ 予約の権限修正

- 役割境界を確立: **設定・権限・プログラムの変更は Claude。gaku-co は実行のみ**(memory 化)。
- VPS 権限正本 `/home/vps-harappa/.claude/settings.json` に **`sns-manager/temp/**` の Write/Edit を私が追加**(他 processor の working と同じ扱い。元は欠落=これが gaku-co の壁の真因)。バックアップ取得済。
- → gaku-co が新文案(火「夏の海は、図鑑です。」/ 土「三浦半島、野比の山で…京急電鉄さん…」)で **予約完走**(ig_job 14/15・fb_job ×2・火6/23 20:00 / 土6/27 8:00)。temp の caption ファイルで文案の正しさを検証済。

### C. lint ERROR / WARN の根治

- **ERROR**: 紛れ込んだ VPS board/pending/README.md を除去 → lint クリーン。
- **WARN**(`scheduled` 規定外): 予約完了の `scheduled` が board 正本に未登録だった(S56「正本×実装ズレ撲滅」の趣旨)。**`scheduled` を「予約完了=終端」として登録**:
  - [board_lint.py](../../garden/services/garden-gaku-co/board_lint.py) VALID_STATUSES に追加
  - [send_pending.py](../../garden/services/garden-gaku-co/send_pending.py) TERMINAL_STATUSES に追加(→ processed/ へ自動片付け)
  - [board/README.md](../../garden/board/README.md) スキーマ・許容値・終端一覧に明記
  - → VPS デプロイ済。lint 違反なし。sns-select board は processed/ へ片付き、pending は承認待ちの recording-sweep のみの正常状態に。
