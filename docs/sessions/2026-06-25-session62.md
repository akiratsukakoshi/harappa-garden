# セッション62(2026-06-25)— Vendor Switch 宿題消化 + Codex read-only/scratch smoke + 移行準備ライン確定

## 発端

ガクチョから「前回の宿題の残タスクをこのまま進めて」と依頼。
途中で目的を再確認し、今回の目的は **今すぐ Codex へ移行することではなく、移行したいタイミングでの移行負荷を下げること** と整理した。

その前提で、やりすぎず、**read-only + temp scratch write までを確証して一旦止める** 方針にした。

## 実装したこと

### 1. bot 側の未対応 engine エラー捕捉

`garden/services/garden-gaku-co/brain/runner.py` に `UnsupportedEngineError` と user-facing message を追加。

`bot.py` / `morning_greet.py` / `night_cheer.py` は `resolve_runner()` の未対応 engine 例外を捕捉し、イベントハンドラ例外や沈黙ではなく、Discord/log に明示メッセージを返すようにした。

### 2. runner 入力面の中立化

Python 側:

- `AgentRunSpec`
- `ToolPolicy`
- `McpSpec`

を追加し、既存 `run(...)` は互換維持しつつ `run_spec(...)` へ寄せた。

JS launcher 側:

- `buildAgentRunSpec()`
- `buildMcpSpec()`
- `buildToolPolicy()`

を追加し、`execute.mcp` / `execute.tool_policy` を runner 内翻訳へ寄せた。

### 3. Codex runner 追加

launcher:

- `engine: codex` を `codex exec` へ翻訳
- 既定は `--sandbox read-only --ephemeral`
- `tool_policy.mode: scratch-write` の時だけ `--sandbox workspace-write`
- `execute.mcp` は Codex runner では未対応として明示エラー

master runner:

- `CodexSubprocessRunner` を追加
- `codex exec --sandbox read-only --ephemeral`
- `tool_policy.mode: scratch-write` で `workspace-write`

### 4. runtime smoke 追加

`garden/runtime/smoke/` を新設。

`run-smoke.py` は temp directory だけに seed/log/state/lock/scratch を作る。
production の board / soil / hmc_tasks には触れない。

対応モード:

- `claude-code` echo smoke
- `codex` echo smoke
- `codex --real-runner` read-only smoke
- `codex --mode scratch-write --real-runner` scratch write smoke

実 Codex CLI の scratch write では、temp scratch 内に `smoke-output.txt` を作成し、内容 `runtime-smoke-ok` を検証した。

### 5. grants host/profile 対応

`garden/runtime/grants.yml` を単一 host 形式から `hosts` / `profiles` 形式へ拡張。

- default: `vps-harappa` / `claude-code`
- 既存 VPS Claude settings は 23 entries 維持
- `local-wsl` / `codex-read-only` profile を追加

`render-claude-settings.py` は `--host` / `--profile` に対応。
単一 host 形式の後方互換も残した。

### 6. 役割語分離

`AGENTS.md` を「Codex=測量士固定」から、セッション冒頭で **測量士 / コーディング担当** を明示する入口へ更新。

`garden/MAP.md` も、S61 宿題の消化状況と残タスクを更新した。

### 7. 測量士の手紙へのコーディング担当応答

`docs/surveyor/letters/2026-06-25.md` の既存 `## コーディング担当の応答` に S62 追記を追加。

S61 時点で未対応だった項目について、今回の実装に合わせて **採用(対応済)** / **更新済** / **保留(次段判断)** を明示した。
