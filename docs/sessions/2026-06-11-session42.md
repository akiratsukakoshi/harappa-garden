## メンション(LINE 仕様の実測と基盤)

- **実測した制約**: ①bot は textV2 + substitution でメンション可 ②**userId 必須**(表示名不可) ③**group/room 宛のみ**(1:1 宛は 400 を実測 → line_push が宛先 prefix C/R 判定で自動テキストフォールバック) ④userId 一括取得 API は認証済アカウント限定 → **発話 webhook から収集**(制約なし)
- **基盤**: line/app.py に収集フック(発話者の userId + 表示名 → `line_collected.json`、本流非干渉)/ soil 運営 5 名に `line_display_name:` 欄新設 / `processor.py sync-line-users` が照合して**全ニックネーム → userId** を `line_users.json` に自動登録
- **残**: メンバー各自がグループで一度発話 → sync 実行 → メンション有効化(それまで「@名前 さん」テキスト表記)

## 天気 on-demand(セッション内の追加依頼 2 件)

1. **任意地点 × 任意日**: 「箱根のあさっての天気」等。地名解決 = venues.json → **国土地理院アドレス検索**(Open-Meteo の地名検索が漢字に弱いことが判明し一段目に採用)→ Open-Meteo(ローマ字)。16 日先まで。出力 = 天気・気温範囲・降水確率・風
2. **風向き追加**: 16 方位、午前/午後それぞれ**最大風速時刻の風向**(現場判断に効くのは最大風速時)。D-2 ブリーフにも自動反映
3. **千葉 = 佐倉フィールド**(ガクチョ確定): venues.json を大多喜仮置き → 佐倉市へ。「佐倉」エイリアスも追加

## 学び・メモ

- **seedling の型が見えた**: 未整理依頼 → Intake 3 問 → 実装資産の棚卸し(HMC 外の単機能ツールにも Legacy Inventory の考え方が使える)→ UX モック先行で粒度確定 → 1 セッションで test。plot_gardener の手順がそのまま効いた
- 「静かに欠落する」型のバグ(morning_greet の Triage)は番人に引っかからない — **人間の違和感が最後の検知器**。条件分岐の else 側に観測性を持たせる設計を今後意識
- `pkill -f` の自己マッチは ssh リモートコマンドでも起きる(S41 の学びの再演。プロセスの実 cmdline を pgrep で確認してからが正解)
- Open-Meteo: 地名検索は漢字に弱い(「箱根」0 件 /「Hakone」OK)→ 日本語地名は国土地理院 API が本命。forecast API はたまに 502(リトライで解消)
- LINE textV2 メンションの「group/room 宛のみ」制約は公式ドキュメントに明記が薄く、実送信の 400 エラー詳細で確定させた
