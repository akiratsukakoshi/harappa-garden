### 1. workflows/ の初期化 = toC 1-a/b/c (おやこ・こども・おとな)

塚越さんが投入した運用テキストをもとに、原っぱ大学の3定期学部の業務フローを **3階層に分解**:

| 階層 | 周期 | ファイル |
|---|---|---|
| (A) 年間→3ヶ月→月次の企画反映 | 年次 + 3ヶ月単位 | [annual-quarterly-planning.md](../../garden/soil/workflows/annual-quarterly-planning.md) |
| (B) 当月の流れ(シフト中心) | 月次 | [monthly-cycle.md](../../garden/soil/workflows/monthly-cycle.md) |
| (C) プログラム実施フロー | 開催毎 | [program-execution.md](../../garden/soil/workflows/program-execution.md) |

時系列で連動:
- (A) で 3ヶ月先までの開催が確定 → (B) で翌々月のシフトが組まれる → (C) で各開催が回る

各ファイルの構造:
- 各ステップに **時期 / 担当 / 動き / アウトプット / 次の遷移** を構造化
- **不変条件 (invariants)** セクションで「守らないと業務が回らない」ルールを明文化
- **種(自律トリガー)候補** セクションで Phase 3 向けの叩き台
