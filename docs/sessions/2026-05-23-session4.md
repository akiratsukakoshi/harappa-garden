## 今日決めたこと(詳細は ADR)

### 大方針

- **3つの時計**: 年/月/開催 が回り、種は各時計の文字盤に植わる
- **種は3形式**: cron / event / state-change
- **役割分担**: 種が点火 → AI が下書き → 庭師(塚越さん)が剪定
- **ガクコ = 庭の出口**(既存の3チャネル + 承認フローを再利用)

### 番人(watcher) の動作

- **定刻 cron + 緊急 push** (Case C)
- 常駐 polling は運用負荷で却下、完全オンデマンドは反応遅すぎで却下
- OS cron で1日数回起動 → 番人が `/queue` を吸う + 種点火チェック
- 緊急種(請求書到着など)のみ別経路で即時 push

### 剪定の置き場

- **重さで自動振り分け** (`pruning_channel` メタデータ)
- 軽(1往復) → `line`(ガクコ `/pending` + LINE personal 通知)
- 中(diff見たい) → `board_with_notify`(garden 連絡板 + LINE 通知)
- 重(議論) → `board`(garden 連絡板のみ)
- **鉄則**: LINE で見えない剪定待ちを garden に放置しない

### ガクコ `core_team` チャネル

- 当面 4名(運営)のまま。**飯田さんは未参加**
- 「企画会議メンバー」と「ガクコ core_team」は別概念
- HMG が core_team へ通知を投げ始める時に再判断
