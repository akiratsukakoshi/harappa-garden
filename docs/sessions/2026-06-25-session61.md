`garden/runtime/` に、ベンダー切替を「単発の移行作業」ではなく Garden Runtime の定期点検として扱うための最小セットを追加。

- `garden/runtime/VENDOR_SWITCH.md`

  - active engine と candidate engine の台帳。
  - 現 active は `seed_runner: claude-code` / `master_runner: claude-code` / `line_provider: anthropic`。

  - Codex 切替リハーサルの手順。
  - precheck → dry-run → read-only run → scratch write → seed smoke → master bot smoke → rollback。
- `garden/runtime/audit-vendor-lock.py`
  - Claude / Anthropic 固有語を検出し、即エラーではなく分類する監査。
  - 分類: `intended` / `runtime-kit` / `live-doc` / `config-surface` / `history` / `unclassified`。
  - 通常監査では `unclassified: 0` を確認。

- `garden/runtime/README.md`

## 次回宿題

未反映の 7 点は、次の順番で進めるのが推奨。

1. **bot 側の未対応 engine エラー捕捉**
   - `GARDEN_GAKU_CO_ENGINE=codex` を入れても沈黙・例外にならず、Discord/log に「未対応」と返る状態にする。
   - 移行実験の観測性を先に確保する。
2. **launcher README の中立化**
   - `claude -p` 前提を runner / engine 語彙へ寄せる。
3. **`AgentRunSpec` / `mcpSpec` / `toolPolicy` の中立化**
   - Python `brain/runner.py` と JS `launcher.mjs` の両方で、Claude CLI 固有の引数名を一段内側に閉じる。
4. **`garden/runtime/smoke/` を追加**
   - 最初は Claude runner で通る smoke を置き、Codex runner 追加時に同じ型で比較できるようにする。
5. **Codex runner 実装 + 実編集なしの初回実走**
   - read-only → scratch write の順。
   - board / soil / hmc_tasks には触らない。
6. **`grants.yml` の host/profile 対応**
   - Codex 実走で権限表現の差分が見えてから広げる。
7. **`MAP.md` / `AGENTS.md` の役割語分離**
   - 実装の形が見えてから、現在地と入口文書を更新する。

次の 1 セッションでは、1〜3 の途中までがちょうどよい。
その次で smoke + Codex runner read-only 実走に入るのが安全。
