## やったこと

### 0. Plaud ブリッジの解決(★最重要発見)
- Plaud MCP(`@plaud-ai/mcp`)の OAuth トークンは `~/.plaud/tokens-mcp.json` に **refresh_token 付きで保存され自動更新**される(パッケージ実装 `refresh_token: data.refresh_token ?? old` を確認)。
- → **そのトークンを持つホスト(ガクチョ認証済の**ローカル WSL**)でなら、ヘッドレス `claude -p` が非対話で Plaud に到達できる**ことを実測 GREEN(`--mcp-config .mcp.json --strict-mcp-config --allowedTools mcp__plaud__* --permission-mode acceptEdits`)。
- refresh_token はサーバー側でローテートし得る → **同一トークンを2ホストで共有すると互いに無効化** → **所有ホストを1つ(ローカル WSL)に固定**(VPS にコピーしない)。
- S53 が挙げた案 (c)(consumer API クライアント)は**不要**になった(MCP トークン自動更新で足りる)。

### 1. launcher に MCP 対応(再利用可能な基盤)
