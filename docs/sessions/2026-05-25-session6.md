## 今日決めたこと(詳細は ADR)

詳細は [ADR 2026-05-25 デイリーワークフローの種化とタスクマスタアーキテクチャ](../decisions/2026-05-25-daily-workflow-and-task-master-architecture.md) の決定 1-7 を参照。

### 決定 1: タスクマスタの置き場 = Obsidian LiveSync + VPS CouchDB

- 平文 MD ミラーを VPS `/opt/garden/tasks/*.md` に保持(seeds が読む)
- Obsidian PC / iPhone は CouchDB に直結 → 数秒で push 同期(更新ボタン不要)
- VPS 不安定対策は Garden 共通課題として並走(restart=always、CouchDB バックアップ、死活監視、Phase 2 番人候補)

### 決定 2: backlog がマスタ、active は派生ビュー

- HMC では Daily が active 直行だったが、HMG では全 recurring が backlog 経由
- active は毎晩クリア、翌朝 backlog から再抽出
- マスタ一本化により「埋もれ漏れ」を構造的に防ぐ

### 決定 3: 種 4 本立て

| 種 | タイミング |
|---|---|
| `daily-pilot/recurring-spawn` | cron 毎日 06:25 |
| `daily-pilot/morning-briefing` | cron 毎日 06:30 |
| `daily-pilot/night-review` | cron 毎日 22:30 |
| `daily-pilot/inbox-process` | event(inbox/*.md 投入) |

### 決定 4: 種の頭脳 = Claude Code ヘッドレス起動 (Garden 全体への適用)

- cron 都度起動(常駐 daemon ではない)
- 既存 SKILL・MEMORY・MCP がそのまま動く
- 種 YAML に `engine:` フィールドでベンダー切替余地

### 決定 5: Triage = LINE + board MD ハイブリッド

- LINE で通知 + 短文返信
- board MD で対話セッション状態 + ログ保持
- 返信ルートは LINE 短文 OR board 直接編集

### 決定 6: night-review = (C) 常に処理

- `[x]` → archive、`[ ]` → backlog 残存、追加 → backlog
- active は必ずクリア、LINE で当日サマリ報告

### 決定 7: `## 追加` 締切なし = 翌日デフォルトで暫定締切付与

- 翌朝 morning-briefing が Triage 候補に自動格上げ
- MD 形式: `- [ ] **{タスク}** (MM/DD締切・暫定)`

## 議論したが結論未定(Phase 3 実装課題)
