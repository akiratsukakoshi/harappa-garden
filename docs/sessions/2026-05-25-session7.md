# 2026-05-25 セッション7 — 種スキーマ起草 + cron 実行ホスト確定 + Phase 3 細分 + VPS secret 管理方針

参加者: 塚越さん(ガクチョー)、Claude (Opus 4.7)

## 開始時の状態

- セッション6 末で次回本命は「種の YAML スキーマ設計 + `shift_manager/monthly-shift-survey` の draft」
- セッション6 で決まっていたこと: デイリーワークフロー4本立て、Claude Code ヘッドレス on VPS、Obsidian LiveSync + CouchDB、Triage = LINE+board ハイブリッド
- 種ファイルの具体構造(スキーマ・配置・実行記述)は draft 化に至っていなかった
- cron 種実行ホストの具体的議論は未着手

## 流れ

1. セッション6 サマリと直近 ADR 3本を読み込み、本命「種 YAML スキーマ + 1本目 draft」を選択
2. スキーマ根幹3点(配置・形式・実行記述)を塚越さんに確認:
   - 配置 = plot 別ディレクトリ
   - 形式 = MD frontmatter + 本文(workflows と揃える)
   - 実行記述 = SKILL 参照 + 自然言語指示
   - **+ 塚越さんから「SKILL 再編問題」が相談事項として提起**
3. `garden/seeds/README.md`(運用ルール・スキーマ定義・相談事項記録)と `shift_manager/monthly-shift-survey.md` を draft 起草
4. 塚越さんから本質的指摘: **「発火タイミング = ローカル PC が落ちていたら走らないのでは?」**
5. 「立ち上がっている状態」と「PC 起動確認したくない」前提を再定義:
   - PC 運用: 夜は落とす(今と同じ)
   - VPS への credentials 配置: OK だがセキュリティ議論したい
6. 論点を A(今確定)/B(本セッションで結論)/C(別セッション)に分類:
   - A → A を先確定 → B へ
7. **A 確定**:
   - cron 種の実行ホスト = すべて VPS(PC 非依存)
   - 種を HMC 依存度で分類(Garden 内完結 vs HMC 依存)
   - Phase 3 を 3a(VPS完結種)/ 3b(HMC移植+secret)/ 3c(HMC依存種active化) に細分
   - 種ファイル群と MAP.md に反映
8. **B 議論(secret 管理)**:
   - 保管: 平文 env + 600 で開始、age 暗号化への余地残す
   - 隔離: サービス単位 Docker 分離、コンテナ内 root 不使用、secret は必要コンテナのみマウント
   - OAuth scope: 「最小化・人事労務 freee 独立 client・読取/書込分離」の大枠合意(詳細別セッション)
   - rotation: 発覚時 + 年1強制ローテ、初期チェックリスト、docs/security/incidents/ に都度記録
9. セッション末処理(ADR 2本、docs/security/README.md §6 追記、本サマリ、MAP.md 仕上げ)

### 種スキーマ・配置・実行記述

- スキーマ = MD frontmatter 9要素 + 本文(workflows と同じ形式)
- 配置 = `garden/seeds/{plot}/{name}.md`
- 実行記述 = SKILL 参照 + 自然言語 prompt(Claude Code ヘッドレス前提)
- SKILL 再編問題は「相談事項」として seeds/README.md と MAP.md に記録 → 種2本目以降で具体化したら ADR 化

### cron 実行ホスト

- すべて VPS 上で起動(PC 非依存)
- セッション6「種の頭脳 = Claude Code ヘッドレス on VPS」の延長線上の再確認

### Phase 3 の細分

| Phase | 内容 |
|---|---|
| 3a | 種ランチャー(VPS) + Garden 内完結種(daily-pilot 4本) active 化 |
| 3b | HMC の VPS 移植 + secret 管理設計 |
| 3c | HMC 依存種(`monthly-shift-survey` 含む) active 化 |

→ 本日 draft 起草の `monthly-shift-survey` は **Phase 3c 待ち**。先に動かせるのは daily-pilot 系。

### VPS secret 管理方針

- 保管: **平文 env + 600 で開始**(WSL ルール VPS 拡張)、age 暗号化への発展余地を残す
- 隔離: **サービス単位 Docker 分離**(gaku-co5.0 / hmc-on-vps / garden-seed-launcher / CouchDB)、コンテナ内 root 不使用、secret は必要コンテナのみマウント
- OAuth scope: 「最小化・人事労務 freee 独立 client・読取/書込分離」の大枠合意(実雅詳細は Phase 3b 別セッション)
- rotation: **発覚時 + 年1強制ローテ**、初期チェックリスト → スクリプト化、docs/security/incidents/ に都度記録

## 議論したが結論未定(Phase 3b 別セッション)

- VPS ハードニング監査・強化(SSH/firewall/fail2ban/自動更新の現状確認)
- HMC OAuth scope の棚卸し
- 人事労務 freee 独立 OAuth client 作成計画
- 読取/書込トークン分離計画
- Docker Compose スケルトン
- ローテスクリプトの設計
- age 暗号化への移行条件
- HMC の VPS 移植 or 必要部分切り出しの具体手順

## 議論したが結論未定(Phase 3a)

- VPS CouchDB + LiveSync セットアップ手順
- 種ランチャー実装(VPS cron → `claude -p` + ログ + on_failure)
- 平文 MD ミラー daemon 実装
- watcher daemon 実装(event 種用)
- 連絡板(`garden/board/`)構造設計(配信本文セクション切り出し規約)
- gaku-co5.0 側「LINE 返信 → board MD 書き戻し」処理

## 議論したが結論未定(本種固有)

- 月次シートの Q列チェック運用(誰がいつ入れるか)
- monthly-shift-survey の発火時刻(暫定 08:00 / morning-briefing と統合か別か)
- computed_inputs の評価主体(ランチャー vs Claude Code)
