## 1. 記憶設計の見落とし(S20 §5 への再検討)

S20 §5「永続記憶 — gaku-co5.0 を移植」では「ガクチョの要件と 90% 一致、移植する」と整理したが、ガクチョから「**そのままにガクコの仕組みを移行するのではなくガーデンの soil に最適化していく必要がある**」との指摘 → 構造軸の本質的差を見落としていたことが判明:

| | gaku-co5.0 memory | Garden soil |
|---|---|---|
| 分類軸 | **scope(チャネル)軸** — MASTER / LINE_STAFF / discord_* / aibou_* | **意味(対象)軸** — people / business / events / workflows / clients |
| 起点 | 対話(誰と話したか) | 業務対象(何についての情報か) |
| 維持役 | 夜間バッチ(processor.py) | 菌糸(S20 で新設、未実装) |

そのまま移植すると memory(scope 軸)と soil(意味軸)が **別の世界として並存** してしまう。ガクチョの意図は逆で:

- 対話で出てきた事実が **意味に応じて soil に流入** する
- **チャネル間漏洩は厳密に防ぐ**(マスターのやり取りが staff_all に流れてはいけない)
- ガクコ宛でない 3者間会話(スタッフ同士の雑談・連絡)も **整理して保持** する

これは S20 で立てた **菌糸 Mode 1 Ingest** の責務と完全に重なる(あのとき「次ソース → soil 反映 + index 追記」と書いたまま接続先を決めていなかった)。

## 2. 記憶の三層分離 ADR(S22 主題)

[2026-05-31-memory-three-layer-and-soil-routing.md](../decisions/2026-05-31-memory-three-layer-and-soil-routing.md) として正式 ADR 化。

| 層 | 入れる対象 | 分類軸 | 漏洩制御 |
|---|---|---|---|
| **RAW** | 対話の生ログ(bot 宛 / 非宛問わず全発話) | scope(チャネル)軸 | scope 間で **絶対に交わらない**(物理ディレクトリ分離 + 読み取り権限分離) |
| **SOIL** | 「事実」だけ(誰・いつ・何・どう) | 意味(対象)軸 | master のみ完全読み取り、下位 scope は **投影ビュー** 経由で限定的に読む(Stage D で実装) |
| **MEMORY WIKI** | 「判断・評価・意図・ニュアンス」 | scope(チャネル)軸 | scope 間で **絶対に交わらない**。master memory は master bot のみ、staff memory は staff bot のみ |

**ポイント: soil には事実しか書かない**。判断ログ・人物評価・戦略意図は scope ローカルの memory wiki に隔離する。

### 振り分け規約(菌糸 Mode 1 Ingest の責務)

| 抽出内容 | 出力先 | 例 |
|---|---|---|
| **事実**(検証可能) | **soil**(該当する対象軸) | 「慶ちゃん来週シフト入れない」→ `soil/people/staff/鈴木慶.md` |
| **判断・評価・意図**(主観的) | **scope memory wiki** | 「慶ちゃんの来月の役割を見直したい」→ `memory/master/wiki/staff_assignment.md` |
| **決定事項**(具体的に動く) | **scope memory wiki** + **soil** | 「6/15 集合は 7 時駅前」→ master memory + `soil/events/{event}.md` |
| **予定**(未来の事実) | **soil** + scope memory(注記レベル) | 同上 |
| **グレーな約束 / 文脈ノイズ** | 捨てる | S20 §5 庭師方針「明示的タスク化が境界」 |

### 3者間会話の扱い

LINE staff グループでスタッフ同士の会話・連絡を:

- **RAW logging**: bot 宛 / 非宛区別なく `scope=line_staff` の RAW に全発話を保持
- **EXTRACT**(夜間バッチ): 事実 / 決定 / 予定を抜き出す
- **出力先**:
  - 公開事実 → soil(events / projects)
  - 進行中の運営感覚・空気感 → `memory/line_staff/wiki/{topic}.md`(scope ローカル、主題別章立て)
- **bot 応答ポリシー**: 発話には反応せず **傍聴のみ**、**@ガクコ明示時のみ介入**
- **整理粒度**: 時系列議事録ではなく **主題別章立て** + `index.md` で主題一覧

### 漏洩防御 — 投影ビュー(Stage D)

| 層 | master bot | staff bot(将来) | aibou bot(将来) |
|---|---|---|---|
| `memory/master/` | 読み書き | 不可視 | 不可視 |
| `memory/line_staff/` | 読み取り(Stage D) | 読み書き | 不可視 |
| `memory/line_aibou_{id}/` | 読み取り(Stage D) | 不可視 | 読み書き(自分のみ) |
| `soil/` | 完全読み取り | 投影ビュー経由 | 投影ビュー経由 |

**投影ビュー**: soil 本体から scope に応じた公開可能ビューを菌糸が生成。Stage A〜C はガクチョ単独運用で漏洩リスクなしのため不要、Stage D(LINE 統合 = チーム公開)直前に実装。

### Stage 段階

| Stage | 内容 | 連動する統合 ADR Stage |
|---|---|---|
| **A** | `memory/master/raw/` で Discord 対話の RAW logging だけ開始 | 統合 Stage 1 |
| **A.5** | 菌糸 Mode 1 Ingest 最小実装(RAW → soil + master memory wiki) | 統合 Stage 2 |
| **B** | 夜間バッチ(EXTRACT + CONSOLIDATE)+ 14 日経過 RAW 削除 | 統合 Stage 3 |
| **C** | bot 起動時に master index + 最近 RAW を context にロード | 統合 Stage 3 |
| **D** | LINE 統合 + 下位 scope RAW + 投影ビュー + マスター透視権 + 3者間会話の EXTRACT | 統合 Stage 4〜7 |

## 3. garden-gaku-co 統合方針 ADR

## 6. 確認の設計(ガクチョ提示の3論点に対する合意)
