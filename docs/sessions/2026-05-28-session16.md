# 2026-05-28 セッション16 — gaku-co を Garden の対話層に統合 + garden-gaku-co 立ち上げ(夜のレポート + 喋るガクコ)

参加者: ガクチョ(塚越暁)、Claude (Opus 4.7)

> セッション15(同日午前: cron 無人実証 + daily-pilot active 化)の続き。
> ガクチョの「gaku-co のユーザー体験を議論したい」から始まり、**Garden の対話層の設計確定 + 実装**まで到達した大きめのセッション。

## きっかけと核心

daily-pilot が無人で回るようになった一方、ガクチョは「**hmc_pilot にあった秘書との対話(朝は対話で整理される / 夜はねぎらいの一言)が、自律化の過程で失われた**」と気づいた。種(claude -p)は孤独なファイル操作用で、対話・記憶・グループ作法を持たない。

議論を通じて、gaku-co5.0 が単なる LINE パイプではなく **会話エージェント**(2段階発言・3層記憶・チャネル別ペルソナ・承認ゲート・情報境界)であり、ガクチョ自身「gaku-co は **Garden の前段階の設計**」と捉えていたことが判明 → **思想統合に躊躇なし**となった。

## 作ったもの — [garden/services/garden-gaku-co/](../../garden/services/garden-gaku-co/)

最小ネイティブ実装。**段1-3 を一晩で稼働**:

| 段 | 内容 | 実装 |
|---|---|---|
| 1 | Discord 送信(疎通) | `send.py`(REST のみ・依存ゼロ) |
| 2 | **夜のレポート** | `night_cheer.py` + cron 22:40 |
| 3 | **喋るガクコ(会話)** | `bot.py`(discord.py 常駐 gateway)+ keepalive |

### 夜のレポート(`night_cheer.py`)= 振り返り処理の完了レポート

ガクチョの要望で「ポエム一言」から **「タスク振り返り処理の完了レポート + 件数 + 正常/異常確認 + ひとことコメント」** に再設計:

```
タスクの振り返り処理を実行しました（2026/05/28 (木)）
完了 6件 / 持ち越し 2件 / 追加 2件 / 期限超過(明日) 2件

（ガクコのひとことコメント）
```

- night-review ログ(`garden/log/{today}-night-review.log`)の `exit_code` と `==NOTIFY==` ブロックを読む
- **件数・エラー検出は決め打ち(claude 非依存=確実)**、コメントだけ claude(ペルソナ)
- `exit_code≠0` / NOTIFY 欠落 → `⚠️ 問題があったかも + ログパス`(=**ヘルスチェック兼用**。エラーがここで分かる)

### 喋るガクコ(`bot.py`)

- discord.py 常駐 gateway → **Discord オンライン表示 + 受信して会話**
- on_message → `claude -p --system-prompt <persona> --strict-mcp-config --disallowedTools <全部> --model sonnet`
- 文脈(active_tasks / board/triage)を Python が読んで prompt 注入(**v1 read-only**、タスク更新は次段)
- 直近12発話をプロセス内で保持(永続記憶は段4)

### ペルソナ — [persona/g-gaku-co.md](../../garden/services/garden-gaku-co/persona/g-gaku-co.md)

- 名: **G-gaku-co**(ガクコシリーズの Garden 版)。呼びかけ「ガクコ / gaku-co」に反応
- **中性的・偏りなく理知的**。tuning で**自然な口語**寄りに緩めた(初版は固かった)
- ガクチョ呼称(音引きなし・「塚越さん」は使わない)→ **全プロジェクト共通**で `~/.claude/CLAUDE.md` に記憶(Claude 自身も以降ガクチョと呼ぶ)

## デプロイ(VPS)

- 配置: `~/garden/services/garden-gaku-co/`、dev-flow(b)= rsync + ホスト実行
- **sudo 不可 / `python3-venv` の ensurepip 無し** → `python3 -m venv --without-pip` + **get-pip でブートストラップ**して discord.py 2.7.1 導入
- 常駐は systemd 不可(sudo 無し)→ **cron キープアライブ**(`*/2 * * * *` + `@reboot` が `run-bot.sh`、pidfile で二重起動防止)
- crontab に night-cheer 22:40 追加。今夜の自動セット: **06:25 / 06:30 / 22:30 / 22:40 / `*/2`+`@reboot`**
- [vps/cron/crontab.snapshot](../../vps/cron/crontab.snapshot) 更新(従来 5/26 のまま古かった → 現状に同期)

## tuning(ライブで詰めた)

| 指摘 | 原因 | 対応 |
|---|---|---|
| トーンが固い | ペルソナが事務的すぎ | 自然な口語・畳みかけない、に緩めた → OK |
| 日付がずれた(5/29) | 22:30 night-review 後の active_tasks が翌日テンプレ + `--system-prompt` で日付が claude に注入されない | **実時刻(JST)を prompt に明示注入** → 「それは明日の予定」と正しく解釈 |
| 「提案」が唐突 | morning-briefing の Triage(board)を bot が surface | 仕様(本物の宿題)。「いきなり畳みかけない」に調整。proactive surfacing の可否は**保留(下記宿題)** |
| 夜レポートが件数取得失敗 | ログに NOTIFY が2つ(prompt テンプレ + claude 実数値)、最初のテンプレを拾っていた | **数字が入った最後のブロックを採用** → 6/2/2/2 正常表示 |
