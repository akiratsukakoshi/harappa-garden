21. PC ↔ iPhone 両方向同期テスト OK(ほぼリアルタイム)

## 今日決めたこと(主要)

→ ADR 全文: [docs/decisions/2026-05-25-couchdb-livesync-implementation.md](../decisions/2026-05-25-couchdb-livesync-implementation.md)

| 決定 | 一行 |
|---|---|
| **vault 戦略** | 既存 `gakuchovault` をそのまま LiveSync 化(Remotely Sync は無効化、Dropbox は二重バックアップとして放置) |
| **Docker network** | CouchDB が `proxy-manager_default` に external 参加する 2NIC 方式(NPM 側 compose は触らない) |
| **E2EE オン + 各種 Obfuscation OFF** | VPS daemon(Phase 3a A-3)が平文 MD を扱える前提で path / properties 難読化は採らない |
| **Chunk size 60** | V3 Rabin-Karp + Self-hosted CouchDB 推奨 |
| **Case-Sensitive OFF** | Windows / iPhone 両端が case-insensitive のため |
| **iPhone は別 vault `gakuchovault-ls` で Fetch** | 既存 vault に手を加えず、ダメージ 0 で動作確認 |
| **端末追加は Setup URI 方式** | passphrase 入力ミス防止 + Wizard の方向選択ミスを根本回避 |
| **docker-compose v1.29 互換** | `version: '3.8'` + `-p garden-couchdb` |
| **NPM パスワードリセット手順** | `bcrypt`(`bcryptjs` ではない)+ ハッシュ長 assertion 必須 |

## 達成感

塚越さんの当初宣言「**A-2 をやってみよう。今日から使えるので**」を素直に達成。

- 既存の `hmc_tasks/` をそのまま LiveSync に乗せたので、Phase 3a 種が CouchDB 経由で触る素地が完成
