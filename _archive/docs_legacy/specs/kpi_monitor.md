# App Definition: KPI Monitor

## 1. アプリ概要
**KPI Monitor** は、Freee会計のデータ（月次推移試算表）をAPI経由で取得し、Google Spreadsheets 上に経営判断に資する高度な「部門別損益計算書（PL）」を自動構築するアプリケーションです。
標準のFreeeレポートでは閲覧しにくい「部門ごとの詳細な損益」と「全社の部門別合算」を、視認性の高いスプリットレイアウトで可視化します。
また、会計年度を跨いだ「昨対比（24ヶ月表示）」を標準機能として提供します。

## 2. 業務フロー (Workflow)

完全自動化されたバッチ処理として動作します。

1.  **Input (Freee):**
    *   日常の会計処理（仕訳登録）が完了していることが前提。
2.  **Process (KPI Monitor):**
    *   `python apps/kpi_monitor/main.py [YYYYMM]` を実行。
    *   指定された開始年月（当期首）に基づき、**昨年度（12ヶ月）** ＋ **当年度（12ヶ月）** の計24ヶ月分のデータを取得。
    *   「部門」タグを持つデータを抽出し、設定ファイルに基づきグルーピング（事業部単位など）。
3.  **Output (Google Sheets):**
    *   指定されたスプレッドシートの各タブ（事業部シート、全社ダッシュボード）を**全上書き更新**。
    *   各シートは「上部：サマリ」「下部：勘定科目詳細」の2段構成。

## 3. 実装要件

### 3.1 ディレクトリ構造
```text
apps/kpi_monitor/
 ├── main.py              # メイン処理 (Fetch -> Build -> Sync)
 ├── sheet_syncer.py      # Google Sheets 同期ロジック (Matrix Update)
 ├── initialize_sheets.py # シート初期化（リセット）スクリプト
 ├── delete_dashboard.py  # シート削除スクリプト
 └── config.py            # 設定 (部門ID定義、科目マッピング)
```

### 3.2 データロジック

*   **2ヵ年取得 (Dual-Year Fetch):**
    *   引数 `YYYYMM` (例: 202510) を当期首として解釈。
    *   自動的に `YYYY-1` 年度のデータも取得し、[昨年度] [当年度] の順にカラムを構成。
*   **スプリットレイアウト (Split Layout):**
    *   **Top Section:** 部門（Section）ごとの主要指標（売上、原価、粗利、販管費、営利）のみを表示。
    *   **Bottom Section:** カテゴリ（事業部）全体での勘定科目別詳細（売上～営業利益まで）を表示。
*   **未分類 (Unallocated) の算出:**
    *   `Unallocated = Total_Company - Sum(Mapped_Departments)`
    *   設定ファイル (`config.py`) に定義されていない部門（Orphans）があった場合も、自動的に「未分類」として吸収され、全社合計との整合性が保たれるロジックを採用。
*   **COGS (売上原価) 対応:**
    *   Freeeの仕様上、「当期商品仕入」「期首商品棚卸」「他勘定振替高」などに分散する科目を全て「売上原価 (COGS)」グループとして集約。

### 3.3 設定 (`config.py`)
*   **DEPT_CATEGORIES:** スプレッドシートのタブ（事業部）と、それに紐づくFreee部門IDの定義。
*   **SECTION_RENAMES:** 視認性を高めるための部門名の別名定義（例: `ID:12345` -> `大阪イベント`）。
*   **KPI_MAPPINGS:** Freeeのカテゴリ名（"売上高" 等）と内部ロジックのマッピング。

### 3.4 検証 (Validation)
*   **Checksum:** 全月の「全社営業利益」と「各事業部営業利益 + 未分類営業利益」の合計が一致することをプログラム内で自動検証。不一致の場合はログにERRORを出力し、同期処理を中断しない（ログ確認用）。
