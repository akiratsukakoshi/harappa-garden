# User Manual: KPI Monitor

## 1. はじめに
このアプリケーションは、Freee会計のデータをもとに、HMC（Harappa Management Cockpit）スプレッドシート上の「部門別損益ダッシュボード」を自動更新するためのツールです。
ボタン一つで、昨年度と当年度の2年分の数値を最新化します。

## 2. 事前準備
作業はターミナル（コマンドライン）で行います。
まず、プロジェクトのディレクトリに移動し、仮想環境を起動してください。

```bash
# プロジェクトフォルダへ移動
cd ~/harappa-cockpit

# 仮想環境の起動
source venv/bin/activate
```

## 3. 月次更新の手順 (Routine Update)
月次決算が完了した後、以下のコマンドを実行してダッシュボードを更新します。
このコマンドは、指定した期（当期）とその前期の**2年分全てのデータを再取得**し、シートを上書き更新します。

### コマンド形式
`python apps/kpi_monitor/main.py [当期の開始年月]`

### 実行例
当期が **2025年10月** 始まりの場合：

```bash
python apps/kpi_monitor/main.py 202510
```

実行するとログが流れ、数分で処理が完了します。「Sync Complete」と表示されれば成功です。
スプレッドシートを開き、数値が更新されていることを確認してください。

## 4. 設定の変更 (部門の追加・変更)
新しい部門ができたり、シートの構成を変えたい場合は、設定ファイルを直接編集します。

**編集ファイル:** `apps/kpi_monitor/config.py`

### よくある変更

#### 部門の名称を変えたい（表示名のみ）
`SECTION_RENAMES` リストにIDと新名称を追加します。

```python
SECTION_RENAMES = {
    1424131: "大阪_イベント", # 左: Freeeの部門ID, 右: シート上の表示名
    9999999: "新規事業部",
}
```

#### 新しい事業部シートを追加したい
`DEPT_CATEGORIES` に新しい定義を追加します。
※ここに追加すると、次回更新時に自動的に新しいタブが作成されます（既存シートの削除が必要な場合があります）。

## 5. トラブルシューティング

### シートのレイアウトが崩れた / 列が足りない
年度が変わって列数が増えた場合、自動で拡張されないことがあります。
その場合はスクリプトでシートを初期化（再作成）できます。

**注意:** これを行うと、シート上の手書きメモなどは消去されます。

```bash
python apps/kpi_monitor/initialize_sheets.py
```

### Validation FAILED と表示される
ログに `Validation FAILED` と出る場合、Freeeから取得した「全社合計」と「各部門の積み上げ合計」が一致していません。
多くの場合、プログラムのロジックが未対応の特殊な勘定科目が追加されたことが原因です。エンジニアに連絡してください。
現在のバージョンでは、未定義の部門があっても「Unallocated（未分類）」に自動吸収されるため、部門漏れによるエラーは発生しません。
